# 滑动窗口最大值

> [LeetCode 239. Sliding Window Maximum](https://leetcode.cn/problems/sliding-window-maximum/)

## 题目

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到最右侧。你只可以看到在滑动窗口内的 k 个数字。返回每次滑动窗口中的最大值。

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
```

## 解答

```java
public static int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] res = new int[n - k + 1];
    Deque<Integer> dq = new ArrayDeque<>(); // 存下标，保持递减

    for (int i = 0; i < n; i++) {
        while (!dq.isEmpty() && dq.peekFirst() < i - k + 1) dq.pollFirst(); // 过期
        while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) dq.pollLast(); // 维护递减
        dq.offerLast(i);
        if (i >= k - 1) res[i - k + 1] = nums[dq.peekFirst()];
    }
    return res;
}
```

## 复杂度

- **时间**：O(n)
- **空间**：O(k)

## 踩坑记录

> 双端队列 `Deque` 做单调队列：`peekFirst/pollFirst` 头部操作，`peekLast/pollLast/offerLast` 尾部操作。

## 复习标记

> 🔁 2026-05-13 思路遗忘——单调递减双端队列（存下标）。每轮三步：①队头过期清理 ②队尾比当前小的全踢（它们永远当不了最大值） ③入队，队头即答案
> 🔁 2026-05-14 复习仍不熟。重点：队头为什么一定是最大值？——第②步while循环把队尾比当前小的全踢了，所以从头到尾递减，队头最大；peekFirst看队头(最大值下标)，peekLast看队尾，pollFirst从头部扔过期，pollLast从尾部踢小的，offerLast从尾部入队
> 🔁 2026-05-20 代码3处错误（均为同一问题：队列存的是索引不是值）：①结果数组大小应为`new int[n-k+1]`不是`new int[n]`；②比较时`nums[i]>=deque.peekLast()`应为`nums[i]>=nums[deque.peekLast()]`，要比索引对应的值；③取结果`res[...]=deque.peekFirst()`应为`res[...]=nums[deque.peekFirst()]`，要取索引对应的值
