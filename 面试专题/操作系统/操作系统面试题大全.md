# 操作系统面试题大全

> 综合整理自 小林coding、javabetter、JavaGuide、牛客、知乎等平台 2024-2025 年高频面试题。
> 标注频率：🔥高频 / 🔥🔥必考 / 🔥🔥🔥核心必背

## 一、操作系统基础

### 1. 什么是操作系统？🔥🔥 ⭐

操作系统是管理计算机硬件和软件资源的**中间层系统**，向上为应用程序提供统一接口，向下管理 CPU、内存、磁盘等硬件资源。

**核心功能**：
- **进程管理**：创建/终止进程、调度、同步与通信
- **内存管理**：内存分配/回收、虚拟内存、内存保护
- **文件管理**：文件创建/删除/读写、目录组织
- **设备管理**：设备驱动、I/O 调度
- **安全与权限**：用户认证、访问控制

---

### 2. 用户态和内核态？🔥🔥🔥 ⭐⭐

| 维度 | 用户态 | 内核态 |
|------|--------|--------|
| **权限** | 受限，不能直接访问硬件 | 完全权限，可执行所有指令 |
| **运行内容** | 用户程序 | 操作系统内核 |
| **地址空间** | 用户空间 | 内核空间 |
| **异常影响** | 只影响当前进程 | 可能导致整个系统崩溃 |

**切换方式**：
- **系统调用**（主动）：如 read()、fork()、mmap()
- **异常/中断**（被动）：如缺页中断、除零异常
- **外设中断**（被动）：如键盘输入、网卡数据到达

> **系统调用**是用户态进入内核态的**唯一合法途径**。

---

### 3. 并发和并行的区别？🔥🔥 ⭐

| 维度 | 并发 | 并行 |
|------|------|------|
| **定义** | 一段时间内多个任务交替执行 | 同一时刻多个任务同时执行 |
| **CPU** | 单核即可（时间片轮转） | 需要多核/多 CPU |
| **本质** | 宏观同时，微观交替 | 物理上的同时 |

## 二、进程与线程

### 4. 进程和线程的区别？🔥🔥🔥 ⭐⭐⭐

| 维度 | 进程 | 线程 |
|------|------|------|
| **本质** | 资源分配的基本单位 | CPU 调度的基本单位 |
| **地址空间** | 独立（隔离） | 共享（同一进程内） |
| **上下文切换** | 开销大（切换地址空间） | 开销小（共享地址空间） |
| **通信** | 需要 IPC 机制 | 直接共享内存 |
| **创建销毁** | 开销大 | 开销小 |
| **崩溃影响** | 不影响其他进程 | 可能导致整个进程崩溃 |

---

### 5. 进程、线程、协程的区别？🔥🔥 ⭐⭐

| 维度 | 进程 | 线程 | 协程 |
|------|------|------|------|
| **调度** | OS 调度 | OS 调度 | 用户态调度 |
| **切换开销** | 大 | 中 | 极小 |
| **地址空间** | 独立 | 共享进程空间 | 共享线程空间 |
| **并发数** | 数百~数千 | 数千~数万 | 数十万~数百万 |
| **编程复杂度** | 低 | 中（需同步） | 高 |

> **协程优势**：用户态调度，不需要内核参与切换，可以轻松创建数十万个协程。Go 的 goroutine、Python 的 async/await 都是协程实现。

---

### 6. 进程的状态和转换？🔥🔥🔥 ⭐⭐

```
创建(New) → 就绪(Ready) ⇄ 运行(Running) → 结束(Terminated)
                         ↓          ↑
                    阻塞(Blocked) ──┘
```

| 转换 | 触发条件 |
|------|---------|
| 就绪 → 运行 | 被调度器选中 |
| 运行 → 就绪 | 时间片用完 / 被高优先级抢占 |
| 运行 → 阻塞 | 等待 I/O、等待锁、sleep |
| 阻塞 → 就绪 | I/O 完成、锁释放、sleep 到期 |

---

### 7. 什么是僵尸进程和孤儿进程？🔥🔥 ⭐

| 类型 | 说明 | 危害 |
|------|------|------|
| **僵尸进程** | 子进程已终止但父进程未调用 wait() 回收，进程表中残留 PCB | 占用 PID 资源，过多会导致无法创建新进程 |
| **孤儿进程** | 父进程先于子进程退出，子进程被 init(PID=1) 收养 | 无危害，init 会负责回收 |

---

### 8. 进程间通信（IPC）有哪些方式？🔥🔥🔥 ⭐⭐

| 方式 | 说明 | 特点 |
|------|------|------|
| **管道（Pipe）** | 内核缓冲区，半双工 | 匿名管道限亲缘进程，命名管道可任意进程 |
| **消息队列** | 内核中的消息链表 | 有拷贝开销，有格式 |
| **共享内存** | 多个进程映射同一块内存 | ✅ 最快（零拷贝），但需处理同步 |
| **信号量** | 计数器，P/V 操作 | 用于同步和互斥 |
| **信号** | 异步通知机制 | 如 SIGKILL、SIGINT |
| **Socket** | 网络通信端点 | ✅ 可跨机器通信 |

---

### 9. 进程上下文切换的开销？🔥🔥 ⭐

**进程上下文**包含：
- 用户空间：虚拟内存、栈、全局变量
- 内核空间：内核栈、寄存器、程序计数器、PCB

**开销来源**：
1. 保存/恢复寄存器组
2. 切换虚拟地址空间（更新页表基址寄存器，导致 TLB 失效）
3. 刷新 CPU 缓存（Cache miss 增加）

> 线程切换（同进程）比进程切换快，因为不需要切换虚拟地址空间。

---

### 10. 进程调度算法？🔥🔥🔥 ⭐⭐

| 算法 | 类型 | 特点 | 适用场景 |
|------|------|------|---------|
| **FCFS（先来先服务）** | 非抢占 | 简单公平，对短作业不友好 | 批处理系统 |
| **SJF（短作业优先）** | 非抢占 | 平均等待时间最短，长作业可能饥饿 | 批处理 |
| **SRTF（最短剩余时间）** | 抢占 | SJF 抢占版，最优但需要预知执行时间 | 理论 |
| **RR（时间片轮转）** | 抢占 | 公平，时间片太短切换开销大 | ✅ 交互式系统 |
| **优先级调度** | 两者皆可 | 可能导致低优先级饥饿 | 实时系统 |
| **HRRN（高响应比优先）** | 非抢占 | `响应比 = (等待+执行)/执行`，兼顾长短作业 | 批处理 |
| **多级反馈队列** | 抢占 | ✅ 综合最优，兼顾响应时间和吞吐量 | 通用 OS |

## 三、同步与死锁

### 11. 线程间同步方式？🔥🔥 ⭐

| 方式 | 说明 |
|------|------|
| **互斥锁（Mutex）** | 加锁后其他线程阻塞等待 |
| **自旋锁（Spinlock）** | 加锁失败忙等待（CAS），适合锁住时间极短的场景 |
| **信号量（Semaphore）** | P/V 操作控制资源访问数量 |
| **条件变量（Condition）** | 等待-唤醒机制，需配合互斥锁使用 |
| **读写锁** | 读锁共享、写锁独占，适合读多写少场景 |

---

### 12. 什么是死锁？产生条件？🔥🔥🔥 ⭐⭐

**死锁**：两个或多个进程互相等待对方释放资源，导致都无法继续执行。

**四个必要条件**（必须同时满足）：
1. **互斥条件**：资源一次只能被一个进程使用
2. **持有并等待**：持有资源的同时等待其他资源
3. **不可剥夺**：已分配的资源不能被强制回收
4. **循环等待**：进程等待形成环路

---

### 13. 如何预防和避免死锁？🔥🔥🔥 ⭐⭐

**预防（破坏四个条件之一）**：

| 策略 | 破坏条件 | 方法 |
|------|---------|------|
| 一次申请所有资源 | 持有并等待 | 进程运行前申请全部资源 |
| 允许抢占 | 不可剥夺 | 申请不到时释放已持有资源 |
| 资源有序分配 | 循环等待 | 所有资源编号，按序申请 |

**避免（运行时判断）**：
- **银行家算法**：分配资源前先检查是否会导致不安全状态

**检测与恢复**：
- 定期检测死锁（资源分配图）
- 恢复：终止进程 / 剥夺资源

---

### 14. 乐观锁和悲观锁？🔥🔥 ⭐

| 维度 | 悲观锁 | 乐观锁 |
|------|--------|--------|
| **假设** | 冲突一定发生 | 冲突很少发生 |
| **策略** | 读/写都加锁 | 读不加锁，写时检查版本 |
| **实现** | 互斥锁、读写锁 | CAS + 版本号/时间戳 |
| **适用** | 写多读少 | 读多写少 |

---

### 15. 活锁和饥饿？🔥 ⭐

- **饥饿（Starvation）**：某个进程长期得不到所需资源，无法推进（如低优先级进程永远等不到 CPU）
- **活锁（Livelock）**：进程状态在改变但整体无法推进（类比：两人过窄桥互相让路，但总让到同一边）

## 四、内存管理

### 16. 虚拟内存是什么？🔥🔥🔥 ⭐⭐⭐

**虚拟内存**：OS 为每个进程提供一个独立的、连续的虚拟地址空间，通过页表映射到物理内存。

**好处**：
1. **突破物理内存限制**：利用局部性原理 + 磁盘 Swap
2. **解决多进程地址冲突**：每个进程独立虚拟空间
3. **内存安全**：进程间内存隔离，互不干扰

**地址转换**：虚拟地址 → MMU 查页表 → 物理地址

---

### 17. 内存分段和分页？🔥🔥🔥 ⭐⭐

| 维度 | 分段 | 分页 |
|------|------|------|
| **划分依据** | 逻辑段（代码段/数据段/栈段） | 固定大小页（Linux 4KB） |
| **大小** | 不固定 | 固定 |
| **地址空间** | 二维（段号+段内偏移） | 一维（页号+页内偏移） |
| **碎片** | 外部碎片 | 内部碎片 |
| **用户可见** | 是 | 否（对用户透明） |
| **共享/保护** | ✅ 方便 | 有限 |

---

### 18. 多级页表和 TLB？🔥🔥 ⭐⭐

**多级页表**：解决单级页表占用内存过大的问题。

```
32位地址（Linux 两级页表）：
高10位 → 页目录索引 → 页目录项 → 二级页表地址
中10位 → 页表索引   → 页表项   → 物理页框号
低12位 → 页内偏移
```

**TLB（快表）**：CPU 内部的页表 Cache，加速地址转换。
- TLB 命中 → 直接获得物理地址（无需访问内存中的页表）
- TLB 未命中 → 查主存页表，同时更新 TLB

---

### 19. 程序的内存布局？🔥🔥 ⭐

```
高地址
┌─────────────┐
│   栈(Stack)   │ ← 局部变量，向低地址增长，约 8MB
├─────────────┤
│ 文件映射区    │ ← mmap、共享库、共享内存
├─────────────┤
│   堆(Heap)    │ ← malloc/new，向高地址增长
├─────────────┤
│   BSS 段     │ ← 未初始化全局/静态变量
├─────────────┤
│   数据段      │ ← 已初始化全局/静态变量
├─────────────┤
│   代码段      │ ← 程序二进制代码（只读）
└─────────────┘
低地址
```

---

### 20. 堆和栈的区别？🔥🔥 ⭐

| 维度 | 栈 | 堆 |
|------|-----|-----|
| **管理** | 编译器自动管理 | 程序员手动管理（可能泄漏） |
| **分配方式** | 静态分配（函数调用） | 动态分配（malloc/new） |
| **生长方向** | 向低地址增长 | 向高地址增长 |
| **大小** | 较小（约 8MB） | 较大 |
| **速度** | 快（移动指针即可） | 慢（需查找空闲块） |
| **碎片** | 无 | 有 |

---

### 21. 页面置换算法？🔥🔥🔥 ⭐⭐

| 算法 | 说明 | 特点 |
|------|------|------|
| **OPT（最佳置换）** | 淘汰未来最长时间不用的页面 | 理论最优，无法实现（需预知未来） |
| **FIFO** | 淘汰最早进入内存的页面 | 简单，可能 Belady 异常 |
| **LRU** | 淘汰最久未被访问的页面 | ✅ 性能好，实现开销大 |
| **Clock（时钟）** | 环形链表 + 访问位，淘汰访问位为 0 的 | LRU 的近似，开销小 |
| **LFU** | 淘汰访问次数最少的 | 对早期高频数据不友好 |

---

### 22. 缺页中断？🔥🔥 ⭐

**缺页中断**：程序访问的虚拟页不在物理内存中时，触发中断，OS 从磁盘（Swap 区或文件系统）将缺失页调入内存。

**处理流程**：
1. CPU 访问虚拟地址 → MMU 查页表 → 页表项有效位为 0
2. 触发缺页异常，进入内核态
3. OS 从磁盘读取缺失页到物理内存
4. 更新页表项
5. 重新执行引起缺页的指令

---

### 23. 写时复制（Copy-on-Write）？🔥🔥 ⭐⭐

**fork() 创建子进程时**：
1. 只复制父进程的**页表**，不复制物理内存
2. 父子进程共享物理内存页（标记为只读）
3. 当任一方尝试**写入**时，触发写保护异常
4. OS 才复制该物理页，各自拥有独立副本

**优势**：减少 fork 时的内存拷贝开销，节约物理内存。

---

### 24. 内存不足时 OS 如何处理？🔥🔥 ⭐

```
内存不足
  ↓
后台内存回收（kswapd，异步）→ 回收干净文件页/写回脏页/Swap 匿名页
  ↓ 仍不足
直接内存回收（同步，阻塞进程）
  ↓ 仍不足
OOM Killer → 杀死占用内存最多的进程
```

**可回收内存**：
- **文件页**：干净页直接释放，脏页先写回磁盘
- **匿名页**：通过 Swap 写入磁盘

## 五、文件系统

### 25. 硬链接和软链接的区别？🔥🔥 ⭐

| 维度 | 硬链接 | 软链接（符号链接） |
|------|--------|-------------------|
| **inode** | 相同（指向同一 inode） | 不同（独立 inode） |
| **删除源文件** | 不影响（引用计数 -1） | 链接失效（悬空链接） |
| **跨文件系统** | ❌ | ✅ |
| **链接目录** | ❌ | ✅ |
| **创建命令** | `ln source target` | `ln -s source target` |

---

### 26. inode 是什么？🔥 ⭐

**inode（索引节点）**：存储文件元数据的数据结构。

**包含**：文件大小、权限、所有者、时间戳、数据块位置指针等

**不包含**：文件名（文件名存在目录项中，目录项映射文件名 → inode 号）

## 六、I/O 系统

### 27. 零拷贝是什么？🔥🔥 ⭐⭐

**传统 I/O**：4 次上下文切换 + 4 次数据拷贝
```
磁盘 → 内核缓冲区 → 用户缓冲区 → Socket缓冲区 → 网卡
```

**零拷贝（sendfile）**：2 次上下文切换 + 2 次数据拷贝（都由 DMA 完成）
```
磁盘 → 内核缓冲区 → 网卡
```

| 技术 | 上下文切换 | 数据拷贝 | 应用 |
|------|----------|---------|------|
| **mmap + write** | 4 次 | 3 次 | 减少一次拷贝 |
| **sendfile** | 2 次 | 2 次 | ✅ Kafka、RocketMQ 使用 |

---

### 28. 阻塞/非阻塞 I/O、同步/异步 I/O？🔥🔥🔥 ⭐⭐

| I/O 模型 | 说明 |
|---------|------|
| **阻塞 I/O** | 调用 read 后线程阻塞，直到数据就绪并拷贝完成 |
| **非阻塞 I/O** | read 未就绪时立即返回错误，应用轮询 |
| **I/O 多路复用** | 一个线程同时监控多个文件描述符（select/poll/epoll） |
| **信号驱动 I/O** | 数据就绪时内核发 SIGIO 信号通知 |
| **异步 I/O（AIO）** | 调用 aio_read 后立即返回，内核自动完成数据拷贝后通知 |

> **注意**：前四种都是**同步 I/O**（数据拷贝阶段需要应用等待），只有 AIO 是真正的异步。

---

### 29. select、poll、epoll 的区别？🔥🔥🔥 ⭐⭐⭐

| 维度 | select | poll | epoll |
|------|--------|------|-------|
| **数据结构** | fd_set（位图） | 动态数组 pollfd | 红黑树 + 就绪链表 |
| **最大连接数** | 1024 | 无限制 | 无限制 |
| **内核遍历** | O(n) 每次全量 | O(n) 每次全量 | O(1) 事件驱动 |
| **内存拷贝** | 每次调用全量拷贝 | 每次调用全量拷贝 | 只拷贝就绪的 fd |
| **触发模式** | 水平触发 | 水平触发 | 水平触发(LT) / 边缘触发(ET) |

**epoll 核心**：
- **红黑树**：管理所有监听的 fd，增删查 O(logn)
- **事件驱动**：内核通过回调函数将就绪事件加入就绪链表
- **ET 模式**：只在状态变化时通知一次，需一次性读完数据（配合非阻塞 I/O）
- **LT 模式**：只要缓冲区有数据就持续通知

> **C10K 问题**：epoll 是解决一台服务器同时处理 10000 个客户端连接的利器。

### 30. Reactor 模式？🔥🔥 ⭐

| 实现 | 模式 | 说明 |
|------|------|------|
| **Redis 6.0 之前** | 单 Reactor 单进程 | 适合内存操作快的场景 |
| **Redis 6.0 之后** | 单 Reactor 多线程 | I/O 读写多线程，命令执行单线程 |
| **Netty** | 多 Reactor 多线程 | 主 Reactor 接收连接，子 Reactor 处理 I/O |
| **Nginx** | 多 Reactor 多进程 | 子进程竞争 accept，通过锁防止惊群 |

## 七、中断

### 31. 什么是中断？🔥🔥 ⭐

**中断**：CPU 停下当前工作，转去处理突发事件，处理完后返回继续执行。

**分类**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **外部中断（硬件）** | 来自 CPU 外部的设备 | 网卡数据到达、键盘输入、时钟 |
| **内部中断（异常）** | CPU 执行指令时产生 | 缺页、除零、系统调用 |

**内部中断细分**：
- **陷阱（Trap）**：有意的，如系统调用 `int 0x80`
- **故障（Fault）**：可恢复，如缺页异常
- **终止（Abort）**：不可恢复，如硬件故障

---

### 32. 中断的处理流程？🔥 ⭐

```
中断/异常发生
  ↓
硬件自动完成：
  1. 关中断
  2. 保存断点（PC → 栈）
  3. 查中断向量表，获得中断服务程序入口
  ↓
软件执行中断服务程序：
  1. 保存现场
  2. 执行中断处理逻辑
  3. 恢复现场
  4. 开中断
  5. 中断返回（iret）
```

## 八、综合题

### 33. fork() 创建子进程做了什么？🔥🔥 ⭐

1. 为子进程分配新的 PID 和 PCB
2. **复制父进程的页表**（虚拟内存结构相同）
3. 共享父进程的物理内存（标记只读，写时复制）
4. 复制文件描述符表
5. 子进程 fork() 返回 0，父进程返回子进程 PID

---

### 34. malloc 是如何分配内存的？🔥🔥 ⭐

```
malloc(size)
  ├─ size < 128KB → brk() 移动堆顶指针（小分配）
  └─ size ≥ 128KB → mmap() 在文件映射区分配（大分配）
```

**两种系统调用**：
- **brk()**：移动进程堆顶指针，适合小分配，不会立即归还内存给 OS
- **mmap()**：在文件映射区创建独立映射，释放时直接归还给 OS

---

### 35. 用户线程和内核线程的区别？🔥 ⭐

| 维度 | 用户线程 | 内核线程 |
|------|---------|---------|
| **管理方** | 用户空间的线程库 | 操作系统内核 |
| **切换** | 不需要内核参与，快 | 需要系统调用，慢 |
| **阻塞** | 一个线程阻塞整个进程 | 只阻塞当前线程 |
| **多核利用** | 不能利用多核 | ✅ 可以 |
| **常见模型** | 多对一 | 一对一 |

---

### 36. 协程为什么比线程轻量？🔥 ⭐

1. **用户态调度**：不需要内核态切换，不涉及系统调用
2. **极小的栈空间**：线程栈通常 1-8MB，协程栈可小至 2-4KB（按需扩展）
3. **无需内核参与**：保存/恢复上下文由用户代码完成

---

### 37. 聊聊 Linux 的 page cache？🔥 ⭐

**Page Cache** 是 Linux 内核管理磁盘缓存的机制：
- 读文件时，先查 Page Cache，命中则直接返回，未命中则从磁盘读取并缓存
- 写文件时，先写入 Page Cache（标记为脏页），由 pdflush/flush 线程异步写回磁盘

**优势**：利用内存加速磁盘访问（基于局部性原理）

### 40. 生产者-消费者问题如何用信号量实现？🔥🔥 ⭐⭐

```c
// 三个信号量
semaphore mutex = 1;    // 互斥访问缓冲区
semaphore empty = N;    // 空槽位数
semaphore full  = 0;    // 已占槽位数

// 生产者
P(empty);    // 等待空槽
P(mutex);    // 加锁
放入数据;
V(mutex);    // 解锁
V(full);     // 增加已占数

// 消费者
P(full);     // 等待有数据
P(mutex);    // 加锁
取出数据;
V(mutex);    // 解锁
V(empty);    // 增加空槽数
```

> **关键**：P 操作的顺序不能颠倒（先 P 资源信号量，再 P 互斥信号量），否则可能死锁。
