## 一、复杂度分析

### 1. 时间复杂度和空间复杂度？🔥🔥🔥 ⭐

**时间复杂度**：算法执行时间与数据规模的增长关系。

| 复杂度 | 名称 | 示例算法 |
|--------|------|---------|
| O(1) | 常数 | 哈希表查找 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 归并排序、快排（平均） |
| O(n²) | 平方 | 冒泡排序、两层循环 |
| O(2ⁿ) | 指数 | 穷举所有子集 |
| O(n!) | 阶乘 | 全排列 |

**空间复杂度**：算法运行过程中占用的额外内存与数据规模的关系。

**常见规则**：
- 加法：取最大项 `O(f(n) + g(n)) = O(max(f(n), g(n)))`
- 乘法：相乘 `O(f(n) × g(n))`
- 常数系数忽略：`O(2n) = O(n)`

---

### 2. 最好/最坏/平均/均摊时间复杂度？🔥 ⭐

| 类型 | 说明 | 示例 |
|------|------|------|
| **最好** | 最理想情况下的复杂度 | 快排 O(n log n) |
| **最坏** | 最差情况下的复杂度 | 快排 O(n²)（已排序数组） |
| **平均** | 所有情况的期望值 | 快排 O(n log n) |
| **均摊** | 连续操作的平均代价 | 动态数组扩容，n 次 append 均摊 O(1) |

## 二、排序算法

### 3. 十大排序算法对比？🔥🔥🔥 ⭐⭐⭐

| 算法 | 平均 | 最坏 | 最好 | 空间 | 稳定 | 原地 |
|------|------|------|------|------|------|------|
| **冒泡** | O(n²) | O(n²) | O(n) | O(1) | ✅ | ✅ |

| **归并** | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ | ❌ |
| **快排** | O(n log n) | O(n²) | O(n log n) | O(log n) | ❌ | ✅ |

> **稳定排序**：相等元素排序后相对位置不变。冒泡、插入、归并是稳定的。

---

### 4. 快速排序的核心思想和优化？🔥🔥🔥 ⭐⭐⭐

**核心思想**：分治法。选一个 pivot，将数组分为 < pivot 和 > pivot 两部分，递归排序。

```
partition 过程：
[3, 6, 8, 10, 1, 2, 1]  pivot = 6
 ↓                     ↓
[3, 1, 2, 1] [6] [8, 10]
```

**优化方案**：
1. **随机选 pivot**：避免最坏情况 O(n²)
2. **三数取中**：取首、中、尾的中位数作为 pivot
3. **小数组用插入排序**：n < 10~20 时插入更快
4. **三路快排**：分为 < pivot、= pivot、> pivot 三部分，处理大量重复元素
5. **尾递归优化**：减少递归栈深度

---

### 5. 归并排序和快速排序怎么选？🔥🔥 ⭐

| 维度 | 快速排序 | 归并排序 |
|------|---------|---------|
| **时间复杂度（最坏）** | O(n²) | O(n log n) |
| **空间复杂度** | O(log n) | O(n) |
| **稳定性** | ❌ 不稳定 | ✅ 稳定 |
| **原地排序** | ✅ | ❌ |
| **适合场景** | 内存有限、随机访问 | 需要稳定排序、链表排序、外部排序 |

> **Java 的 Arrays.sort()**：基本类型用双轴快排，对象用 TimSort（归并+插入）。

---

### 6. 堆排序的过程？🔥🔥 ⭐

**堆**：完全二叉树，父节点 ≥ 子节点（大顶堆）或 ≤ 子节点（小顶堆）

**堆排序步骤**：
1. 建堆：从最后一个非叶节点开始，自下而上 heapify → O(n)
2. 排序：将堆顶（最大值）与末尾交换，缩小堆范围，重新 heapify → O(n log n)

**Top-K 问题**：用大小为 K 的小顶堆，遍历数据，比堆顶大则替换 → O(n log K)

---

### 7. 排序算法的实际应用？🔥 ⭐

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 小规模数据（< 50） | 插入排序 | 常数因子小，对小数据快 |
| 大规模随机数据 | 快速排序 | 平均性能最好 |
| 需要稳定排序 | 归并排序 / TimSort | 保序性 |
| 内存受限 | 堆排序 | O(1) 额外空间 |
| 外部排序（数据在磁盘） | 归并排序 | 顺序访问友好 |
| 范围小的整数 | 计数排序 | O(n+k)，非比较排序 |

## 三、查找算法

### 8. 二分查找的模板？🔥🔥🔥 ⭐⭐

**前提**：有序数组（或具有单调性/分段单调性）。

```java
// 标准二分查找
int binarySearch(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防溢出
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

// 查找左边界（第一个等于 target）
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return left;
}
```

**常见变体**：
- 查找第一个/最后一个等于 target
- 查找第一个大于/大于等于 target
- 旋转排序数组查找
- 求平方根

**注意**：`mid = left + (right - left) / 2` 防止 `left + right` 溢出。

### 13. BST、AVL、红黑树、B+ 树对比？🔥🔥🔥 ⭐⭐

| 树 | 特点 | 查找 | 插入 | 应用 |
|----|------|------|------|------|
| **BST** | 左 < 根 < 右 | O(n) 最坏 | O(n) | 基础 |
| **AVL** | 严格平衡（高度差 ≤ 1） | O(log n) | O(log n) | 内存数据库 |
| **红黑树** | 近似平衡（最长路径 ≤ 2 倍最短） | O(log n) | O(log n) | ✅ HashMap、TreeMap、C++ map |
| **B+ 树** | 多路平衡，叶子链表 | O(log n) | O(log n) | ✅ MySQL 索引、文件系统 |

> **为什么 MySQL 用 B+ 树不用红黑树？** B+ 树一个节点存多个 key（页），减少磁盘 IO 次数；树矮胖，3~4 层即可存储千万级数据。

## 五、算法思想

### 14. 动态规划（DP）的核心？🔥🔥🔥 ⭐⭐⭐

**核心要素**：
1. **最优子结构**：大问题最优解包含子问题最优解
2. **重叠子问题**：子问题会被重复计算（用备忘录/DP 表避免）
3. **状态转移方程**：f(n) 与 f(n-1) 的关系

**解题步骤**：
1. 定义 dp 数组的含义
2. 推导状态转移方程
3. 确定初始条件和遍历顺序

**经典题目**：

| 题目 | 状态转移方程 | 难度 |
|------|------------|------|
| **爬楼梯** | `dp[i] = dp[i-1] + dp[i-2]` | ⭐ |
| **0-1 背包** | `dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi] + vi)` | ⭐⭐ |
| **最长递增子序列** | `dp[i] = max(dp[j] + 1)` 其中 `nums[j] < nums[i]` | ⭐⭐ |
| **最长公共子序列** | `dp[i][j] = dp[i-1][j-1]+1 或 max(dp[i-1][j], dp[i][j-1])` | ⭐⭐ |
| **编辑距离** | `dp[i][j] = min(插/删/改) + 1` | ⭐⭐⭐ |
| **最大子数组和** | `dp[i] = max(nums[i], dp[i-1] + nums[i])` | ⭐⭐ |

### 15. 回溯算法的模板？🔥🔥 ⭐⭐

**适用**：排列、组合、子集、N 皇后、数独等

```java
void backtrack(路径, 选择列表) {
    if (满足结束条件) {
        结果.add(路径);
        return;
    }
    for (选择 : 选择列表) {
        做选择;
        backtrack(路径, 选择列表);
        撤销选择;  // 回溯
    }
}
```

**经典题目**：全排列（46）、子集（78）、组合（77）、N 皇后（51）、括号生成（22）

**剪枝优化**：
- 排序 + 提前终止
- 去重（used 数组或排序跳过）

### 17. 分治算法？🔥 ⭐

**核心**：将大问题分解为独立子问题，分别求解后合并。

**三步**：分解（Divide）→ 解决（Conquer）→ 合并（Combine）

**经典应用**：

|------|---------|--------|

| **快速排序** | 选 pivot 分区后递归 | O(n log n) |
| **最大子数组和** | 分左右和跨中三部分 | O(n log n) |
| **最近点对** | 分成左右两部分 | O(n log n) |

### 18. BFS 和 DFS？🔥🔥 ⭐⭐

| 维度 | DFS（深度优先） | BFS（广度优先） |
|------|----------------|----------------|
| **数据结构** | 栈（递归） | 队列 |
| **空间复杂度** | O(h)（h=树高） | O(w)（w=最大宽度） |
| **特点** | 一条路走到底 | 逐层探索 |
| **最短路径** | ❌ | ✅ 无权图最短路径 |
| **适用** | 全排列、回溯、连通性 | 层序遍历、最短距离、拓扑排序 |

```java
// BFS 模板
Queue<Node> queue = new LinkedList<>();
queue.offer(start);
while (!queue.isEmpty()) {
    Node cur = queue.poll();
    for (Node next : cur.neighbors()) {
        if (!visited[next]) {
            visited[next] = true;
            queue.offer(next);
        }
    }
}
```

### 26. 如何判断链表有环？找环的入口？🔥🔥 ⭐

**判断有环**：快慢指针，相遇则有环。

**找环入口**：
1. 快慢指针相遇后，将一个指针移到头部
2. 两个指针都以相同速度前进，再次相遇即为入口

```java
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) {  // 有环
        slow = head;
        while (slow != fast) {
            slow = slow.next;
            fast = fast.next;
        }
        return slow;  // 环的入口
    }
}
```

### 27. 反转链表？🔥🔥 ⭐

```java
// 迭代法
ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
```

**变体**：反转前 N 个、反转区间 [m, n]、K 个一组反转

### 29. 合并两个有序链表/数组？🔥 ⭐

```java
// 双指针合并
ListNode merge(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), tail = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { tail.next = l1; l1 = l1.next; }
        else { tail.next = l2; l2 = l2.next; }
        tail = tail.next;
    }
    tail.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

### 30. 二叉树的最近公共祖先？🔥🔥 ⭐

```java
TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;  // p、q 分居两侧
    return left != null ? left : right;
}
```
