## 一、原理深挖

### 1.1 大O表示法的本质

> [!info] 核心定义
> 大O表示法 **不表示真实执行时间**，而是描述 **执行时间随数据规模n增长的趋势（上界）**。
> 它关注的是"量级"，忽略常量、低阶项和系数。

```
时间复杂度排序（从快到慢）：
常 → 对 → 幂 → 指 → 阶

O(1) < O(logn) < O(n) < O(nlogn) < O(n²) < O(2^n) < O(n!)
```

> [!question] 为什么忽略常量和系数？
> 当n足够大时，O(2n)和O(n)的增长趋势一样——都是线性增长。
> 常量项100在n=1000000时微不足道，所以O(100) = O(1)。

### 1.2 常见复杂度与判定方法

#### O(1) — 常数时间

执行时间 **不随n变化**，与数据规模无关。

```java
// 固定100次循环 → O(1)，不是O(100)
for (int i = 0; i < 100; i++) {
    System.out.println(i);
}

// 数组下标直接寻址 → O(1)
int value = arr[5];
```

> [!tip] 误区提醒
> 循环次数是**固定常量**（如100次），复杂度是O(1)，不是O(n)。
> 只有循环次数与n有关时才是O(n)。

#### O(logn) — 对数时间

每次迭代 **翻倍或折半**，n被"压缩"执行。

```java
// 二分查找 — 每次折半 → O(logn)
int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

// 循环变量翻倍 → O(logn)
for (int i = 1; i < n; i = i * 2) {
    System.out.println(i); // 执行 log2(n) 次
}
```

> [!info] log底数不重要
> O(log₂n) = O(log₁₀n) = O(logn)，因为不同底数的对数之间只差一个常系数：log₂n = log₂10 × log₁₀n。

#### O(n) — 线性时间

**一层循环**，执行次数与n成正比。

```java
// 单层遍历 → O(n)
for (int i = 0; i < n; i++) {
    System.out.println(arr[i]);
}

// 顺序查找 → O(n)
int find(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) return i;
    }
    return -1;
}
```

#### O(nlogn) — 线性对数时间

常见于高效排序算法。

```java
// 归并排序、快速排序 → O(nlogn)
// 一层O(n)的外层循环 × 一层O(logn)的分治
```

#### O(n²) — 平方时间

**嵌套循环**，内外层都与n有关。

```java
// 两层嵌套 → O(n²)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        System.out.println(arr[i][j]);
    }
}

// 冒泡排序 → O(n²)
void bubbleSort(int[] arr) {
    for (int i = 0; i < arr.length; i++) {
        for (int j = 0; j < arr.length - i - 1; j++) {
            if (arr[j] > arr[j+1]) swap(arr, j, j+1);
        }
    }
}
```

### 1.3 复杂度判定口诀

> [!tip] 快速判定法
> 1. **看循环层数**：一层→O(n)，二层→O(n²)，三层→O(n³)
> 2. **看循环变量变化**：翻倍/折半→O(logn)，递减到0→O(n)
> 3. **忽略常量、低阶项和系数**：O(3n² + 5n + 100) = O(n²)
> 4. **递归看分叉**：T(n) = aT(n/b) + f(n)，用主定理求解

### 1.4 空间复杂度

> [!info] 类似思路
> 空间复杂度同理，衡量的是 **额外占用内存随n增长的趋势**。
> - O(1)：只用几个变量（原地排序）
> - O(n)：需要与n等长的额外数组
> - O(n²)：二维辅助矩阵

## 二、面试回答框架

### 2.1 核心问题："什么是时间复杂度？"

> [!success] 三段式回答模板
> **是什么（定义）：** 大O表示法描述的是算法执行时间随数据规模n增长的趋势，不表示真实时间。
> **怎么排（排序）：** 常对幂指阶——O(1) < O(logn) < O(n) < O(nlogn) < O(n²) < O(2^n)。
> **怎么看（判定）：** 看循环层数和循环变量变化规律，忽略常量、系数和低阶项。

### 2.2 追问应对

| 追问 | 回答要点 |
|------|----------|
| "怎么分析一段代码的复杂度？" | 看循环层数（一层O(n)，两层O(n²））；看循环变量变化（翻倍/折半→O(logn)；忽略常量、系数、低阶项 |
| "O(1)是不是意味着只执行一次？" | 不是。O(1)表示执行时间**不随n变化**，固定循环100次也是O(1) |
| "O(logn)的底数是多少？" | 底数不重要，O(log₂n) = O(log₁₀n) = O(logn)，因为底数之间只差常系数 |
| "空间复杂度怎么分析？" | 同理看额外开辟的内存随n的变化趋势，O(1)只用几个变量，O(n)需要一个等长数组 |

> [!question] 面试陷阱
> "这个算法循环了100次，复杂度是O(100)还是O(n)？"
> → **O(1)**！循环次数是固定常量，不随n变化。

## 三、跨题串联

> [!tip] 复杂度是理解所有集合性能的"底座"，面试中从复杂度出发可以串联到几乎所有集合题。

### 3.1 向下串联：各集合的复杂度

| 集合 | 查找 | 插入（尾部） | 插入（中间） | 删除 | 关键原因 |
|------|------|-------------|-------------|------|----------|
| **ArrayList** | **O(1)** | O(1)均摊 | **O(n)** | **O(n)** | 数组下标寻址O(1)；中间操作需搬移 |
| **LinkedList** | O(n) | O(1) | O(1)已知位置 | O(1)已知位置 | 链表无下标，遍历O(n)；插入改指针O(1) |
| **HashMap** | **O(1)** 平均 | O(1) 平均 | — | O(1) 平均 | hash定位桶，最坏hash冲突→O(n)（JDK8红黑树→O(logn)) |
| **TreeMap** | O(logn) | O(logn) | — | O(logn) | 红黑树平衡查找 |

### 3.2 串联到具体考点

> [!example] 串联路径
> **复杂度 → ArrayList**：为什么ArrayList查O(1)？→ 数组内存连续 + 寻址公式 `base + index * size`，一步到位。中间插删O(n)？→ `System.arraycopy`搬移元素。
>
> **复杂度 → HashMap**：为什么HashMap查O(1)？→ hash散列直接定位桶。最坏O(n)什么时候？→ 所有keyhash冲突落在同一个桶。JDK8如何优化？→ 红黑树退化为O(logn)。
>
> **复杂度 → 树结构**：为什么红黑树/平衡树是O(logn)？→ 每次比较折半搜索空间，类似二分查找。

### 3.3 串联到算法题

- 二分查找 → O(logn) → HashMap的桶内查找优化
- 动态数组扩容 → 均摊O(1) → ArrayList尾插分析
- 排序算法 → O(nlogn) vs O(n²) → TreeMap内部红黑树的插入维护代价
