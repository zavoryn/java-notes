# 操作系统总览

> [!summary] 模块速记
> **操作系统六大核心：进程管理（进程线程IPC）、线程同步（锁+死锁）、内存管理（虚拟内存+页表+置换）、IO模型（epoll+零拷贝）、底层原理（CPU缓存+文件系统）、综合面试（串联跨题）。**
> **一条面试主线：一个Java请求 → 进程调度 → 内存分配 → IO处理 → 磁盘读写。**

## 一条主线串到底

```
一个 Java HTTP 请求的生命旅程
  │
  ├── ① Tomcat 接收请求：epoll 监听 socket → 线程池分配线程
  │     → [[04-IO与网络模型/IO多路复用|IO多路复用]]
  │     → [[01-进程管理/进程与线程|进程与线程]]
  │
  ├── ② 线程调度：OS 内核将 CPU 时间分配给线程
  │     → 进程调度算法（CFS 完全公平调度）
  │     → 用户态 → 内核态切换开销
  │
  ├── ③ 内存分配：JVM 堆内存 new 对象 → OS 虚拟内存 → 缺页中断分配物理页
  │     → [[03-内存管理/内存管理|内存管理]]
  │     → 虚拟内存 → 页表 → TLB → 物理内存
  │
  ├── ④ 并发安全：多线程竞争共享资源 → synchronized/Mutex → CAS → 伪共享
  │     → [[02-线程同步与死锁/线程同步与死锁|线程同步与死锁]]
  │     → [[05-底层原理与工具/CPU缓存与伪共享|CPU缓存与伪共享]]
  │
  ├── ⑤ IO 读写：Redis 查数据 → epoll + 纯内存 → Kafka 写日志 → sendfile 零拷贝
  │     → [[04-IO与网络模型/IO多路复用|IO多路复用]]
  │     → [[04-IO与网络模型/零拷贝|零拷贝]]
  │
  └── ⑥ 磁盘持久化：MySQL 写 redo log → 顺序写 → PageCache → fsync
        → [[03-内存管理/内存管理|内存管理]]（swap/PageCache）
        → [[05-底层原理与工具/文件系统与Linux|文件系统与Linux]]（inode/磁盘IO）
```
