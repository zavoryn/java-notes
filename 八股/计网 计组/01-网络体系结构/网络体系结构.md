# 网络体系结构

> [!tip] 模块速记
> **OSI 七层是理论标准（宪法），TCP/IP 四层是实际标准（现行法律）。记住 TCP/IP 四层就够面试，但要知道 OSI 的存在。ARP 把 IP 翻译成 MAC，路由器工作在网络层，交换机在数据链路层。**

---

## 一、OSI 七层模型 vs TCP/IP 四层模型

### 1.1 OSI 七层模型

| 层号 | 名称 | 功能 | 比喻 | 典型协议/设备 |
|------|------|------|------|--------------|
| 7 | 应用层 | 为用户提供网络服务接口 | 写信的人和读信的人 | HTTP、FTP、DNS、SMTP |
| 6 | 表示层 | 数据格式转换、加密解密 | 把信翻译成通用语言 | JPEG、ASCII、SSL/TLS |
| 5 | 会话层 | 建立/管理/终止会话 | 确保对话有头有尾，不会说着说着断了 | RPC、NetBIOS |
| 4 | 传输层 | 端到端可靠传输 | 邮局：确保信送到对方手里 | TCP、UDP |
| 3 | 网络层 | 寻址与路由选择 | 邮政分拣中心：决定信走哪条路线 | IP、ICMP、路由器 |
| 2 | 数据链路层 | 相邻节点间可靠传输 | 两个相邻邮局之间的交接手续 | MAC、交换机、网桥 |
| 1 | 物理层 | 比特流传输 | 卡车运邮袋：实际搬运物理介质 | 网线、光纤、集线器 |

### 1.2 TCP/IP 四层模型

| TCP/IP层 | 对应OSI | 比喻 |
|----------|---------|------|
| 应用层 | 应用层+表示层+会话层 | 写信、读信 |
| 传输层 | 传输层 | 邮局保证送达 |
| 网际层 | 网络层 | 邮政分拣中心 |
| 网络接口层 | 数据链路层+物理层 | 卡车+公路 |

### 1.3 五层参考模型（教学常用）

实际教学中常采用五层模型：应用层、传输层、网络层、数据链路层、物理层——既兼顾了 OSI 的清晰分层，又贴合 TCP/IP 的实际协议。

> [!note] 记忆口诀
> **物数网传会表应**（物理→数据链路→网络→传输→会话→表示→应用）

## 二、常见网络设备对比

| 设备 | 工作层级 | 功能 | 比喻 | 关键区别 |
|------|----------|------|------|----------|
| **集线器（Hub）** | 物理层 | 收到信号后广播到所有端口 | 大喇叭——一喊所有人都听到 | 不识别帧，所有端口共享带宽，冲突域大 |
| **交换机（Switch）** | 数据链路层 | 根据 MAC 地址转发帧到目标端口 | 快递员——根据门牌号精准投递 | 自学习 MAC 表，每个端口独立带宽，隔离冲突域 |
| **路由器（Router）** | 网络层 | 根据 IP 地址路由转发到目标网络 | 长途大巴调度站——看目的地决定走哪条高速 | 跨网络转发，隔离广播域，连接不同网段 |
| **网桥（Bridge）** | 数据链路层 | 连接两个局域网段，根据 MAC 过滤 | 两个小区之间的天桥 | 早期设备，现已被交换机取代 |

> [!warning] 面试辨析重点
> - **冲突域**：同一物理线路上可能发生碰撞的范围。集线器所有端口同属一个冲突域，交换机每个端口是独立冲突域。
> - **广播域**：广播帧能到达的范围。路由器隔离广播域（不会转发广播包），交换机默认不隔离（除非划分 VLAN）。

## 三、ARP 协议

### 3.1 ARP 是什么？

ARP（Address Resolution Protocol，地址解析协议）负责将 **IP 地址解析为 MAC 地址**。

在网络中，IP 地址是逻辑地址（用于路由），MAC 地址是物理地址（用于局域网内实际传输）。数据包从一台主机到另一台主机，在局域网内最终靠 MAC 地址交付。

**比喻**：IP 地址像"邮编+门牌号"（告诉你大致在哪条路），MAC 地址像"身份证号"（精确到具体的人）。ARP 就是根据邮编查身份证号的工具。

### 3.2 ARP 工作流程

```
主机A要发数据给主机B（同一局域网）：

① 主机A查本地ARP缓存表
   └─ 有B的MAC？→ 直接封装发送
   └─ 没有？→ 进入步骤②

② 主机A广播ARP请求：
   "谁是 192.168.1.5？告诉 192.168.1.1（我的MAC是 aa:bb:cc:dd:ee:ff）"

③ 所有主机收到广播，只有主机B匹配IP
   └─ 主机B回复ARP响应（单播）：
     "192.168.1.5 的 MAC 是 11:22:33:44:55:66"

④ 主机A收到响应，更新ARP缓存表，用B的MAC封装帧发送
```

> [!tip] ARP缓存
> ARP 解析结果会缓存在本地，有效期通常 20 分钟。可以用 `arp -a` 查看。

### 3.3 ARP 欺骗（ARP Spoofing）

**原理**：攻击者向局域网发送伪造的 ARP 响应，将目标 IP 对应的 MAC 替换为攻击者的 MAC。

**后果**：
- 流量劫持：受害者的数据包发给了攻击者
- 中间人攻击：攻击者可以截获、篡改数据
- 网络中断：将 MAC 映射为不存在地址导致断网

**防御手段**：
- 静态 ARP 表（IP-MAC 绑定，不可被动态更新覆盖）
- DAI（Dynamic ARP Inspection，交换机级检测）
- 端口安全（交换机限制每端口 MAC 学习数量）
- 使用 HTTPS/TLS 加密（即使被劫持也无法解密内容）

## 四、IP协议与IPv4地址分类

### 4.1 IP协议的作用

IP协议工作在**网络层**，核心职责两个：
- **寻址**：为每台主机分配唯一地址（IP地址），标识数据从哪来、到哪去
- **路由**：路由器根据目的IP地址选择转发路径

**🔗 知识串联：**
- **串联 ARP**：[[#三、ARP 协议]] 把 IP 翻译成 MAC——IP 是"门牌号"（逻辑地址，跨网络路由用），MAC 是"身份证号"（物理地址，局域网内投递用）。数据包跨网络传输时，**IP 地址不变，但每一跳的 MAC 地址都会变**
- **串联路由器**：[[#二、常见网络设备对比]] 中的路由器工作在网络层，就是根据 IP 地址做路由转发的设备
- **串联 DNS**：[[07-网络应用技术/DNS与CDN]] 把域名翻译成 IP 地址——DNS 给你门牌号，路由器按门牌号找路
- **串联 HTTPS**：[[06-HTTPS与安全认证/HTTPS与TLS详解]] 的 TLS 证书绑定的是域名/IP，DNS 解析出 IP 后才能建立 TCP → TLS 连接

### 4.2 IPv4 地址分类

```
A类：0开头   | 1.0.0.0 ~ 126.255.255.255    | 网络位8位  + 主机位24位 | 超大型网络
B类：10开头  | 128.0.0.0 ~ 191.255.255.255  | 网络位16位 + 主机位16位 | 中型网络
C类：110开头 | 192.0.0.0 ~ 223.255.255.255  | 网络位24位 + 主机位8位  | 小型网络
D类：1110开头| 224.0.0.0 ~ 239.255.255.255  | 组播地址（不分配给主机）
E类：1111开头| 240.0.0.0 ~ 255.255.255.255  | 保留实验用
```

> [!note] 速记
> **A 类可容纳 1600 万+主机，B 类 6 万+，C 类只有 254 台。日常见的 `192.168.x.x` 都是 C 类。**
> **特殊地址：`127.0.0.1` = 本机回环地址，`0.0.0.0` = 本机所有 IP，`255.255.255.255` = 广播地址。**

### 4.3 子网掩码

**为什么需要？** 传统分类太死板——A 类一个网络 1600 万主机，大部分用不完。子网掩码允许灵活切分。

```
IP:   192.168.1.100
掩码: 255.255.255.0 (= /24)

网络地址: 192.168.1.0     ← 主机位全0，标识这个网络本身
广播地址: 192.168.1.255   ← 主机位全1，向这个网络内所有主机广播
可用主机: 192.168.1.1 ~ 192.168.1.254  ← 254台（2^8 - 2）
```

**CIDR（无类域间路由）：** `/24` 表示前 24 位是网络号，简洁明了。`/16` = 65534 台主机，`/28` = 14 台主机。

### 4.4 私有 IP 与 NAT

```
10.0.0.0 ~ 10.255.255.255       (/8)   ← A类私有
172.16.0.0 ~ 172.31.255.255     (/12)  ← B类私有
192.168.0.0 ~ 192.168.255.255   (/16)  ← C类私有
```

**🔗 串联 NAT：** 私有 IP 在公网不可路由，通过 **NAT（网络地址转换）** 映射为公网 IP。这就是你家里所有设备都是 `192.168.x.x`，但对外只有一个公网 IP 的原因。

> **面试怎么考：** "IP 地址怎么分类？子网掩码干什么的？192.168.1.0/24 有多少可用主机？"
>
> **面试话术：** "IPv4 地址分 A/B/C/D/E 五类，按网络位和主机位划分。但实际使用中通过子网掩码做更灵活的划分——比如 `/24` 表示前 24 位是网络号、后 8 位是主机号，可用主机 254 台（去掉全 0 的网络地址和全 1 的广播地址）。私有 IP 地址（10.x、172.16~31.x、192.168.x）配合 NAT 使用，解决了公网 IP 不够用的问题。"

## 五、ICMP协议

### 5.1 ICMP 是什么？

ICMP（Internet Control Message Protocol，互联网控制报文协议）工作在**网络层**，是 IP 协议的"搭档"——IP 只管发数据不报错，ICMP 负责报告网络问题和诊断。

**🔗 知识串联：**
- **串联 IP 协议**：[[#四、IP协议与IPv4地址分类]] 中提到 IP 没有"反馈机制"——包丢了、路由不可达，IP 本身不会告诉你。ICMP 就是 IP 的**报错通道**
- **串联路由器**：路由器找不到路径时，会发 ICMP "Destination Unreachable"（目标不可达）告诉发送方
- **串联 04 QUIC**：[[04-传输层协议对比/TCP vs UDP]] 中 QUIC 基于 UDP，而 UDP 不生成 ICMP——所以 QUIC 在网络诊断上比 TCP 弱一些
- **串联 08 面试题**：[[08-综合面试题/计网面试突击]] 中的"ping 底层原理"就考这个

### 5.2 ping 原理

```
你: ping www.baidu.com

① 你的电脑发送 ICMP Echo Request（类型8）→ 目标主机
② 目标主机回复 ICMP Echo Reply（类型0）→ 你的电脑
③ 你计算 RTT（往返时间），显示结果

64 bytes from 220.181.38.148: icmp_seq=1 ttl=52 time=12.3 ms
```

**ping 用的是 ICMP，不是 TCP 也不是 UDP。** 它直接跑在网络层之上。

### 5.3 traceroute 原理

```
发 TTL=1 的包 → 第1个路由器：TTL减到0，丢弃，回 ICMP Time Exceeded → 知道路由器1
发 TTL=2 的包 → 第2个路由器：TTL减到0，丢弃，回 ICMP Time Exceeded → 知道路由器2
发 TTL=3 的包 → ...
直到到达目标 → 回 ICMP Echo Reply → 路径完整！
```

**比喻**：traceroute 像逐站查公交路线——第一趟车只坐一站就下车，知道第一站是哪；第二趟坐两站……最终画出完整路线。

> **面试怎么考：** "ping 的原理？用了什么协议？traceroute 怎么工作的？"
>
> **面试话术：** "ping 基于 ICMP 协议，发 Echo Request 等对方回 Echo Reply，测往返时间。traceroute 利用 IP 包的 TTL 字段——发 TTL=1 的包，第一个路由器 TTL 减到 0 就丢弃并回 ICMP Time Exceeded，逐跳递增 TTL 就能探测出完整路径。两者都依赖 ICMP，跑在网络层，不涉及传输层的 TCP/UDP。"

## 六、IPv6 vs IPv4（了解即可）

| 对比维度 | IPv4 | IPv6 |
|----------|------|------|
| **地址长度** | 32 位（约 43 亿个） | 128 位（3.4×10³⁸个，几乎无限） |
| **表示方式** | 点分十进制 `192.168.1.1` | 冒号十六进制 `2001:0db8::1` |
| **头部** | 可变长度（20~60 字节），字段多 | **固定 40 字节**，简化字段，路由处理更快 |
| **NAT** | 需要（地址不够） | 不需要（地址足够每台设备一个公网 IP） |
| **安全性** | IPSec 可选 | **内置 IPSec** |
| **配置** | 手动 or DHCP | **自动配置**（SLAAC） |

**🔗 知识串联：**
- **串联 NAT**：IPv4 地址不够才需要 NAT——IPv6 地址空间足够大，理论上每台设备都能有公网 IP，不再需要 NAT 穿透的问题
- **串联 DNS**：[[07-网络应用技术/DNS与CDN]] 中的 AAAA 记录就是 IPv6 专用记录
- **串联 04 QUIC**：[[04-传输层协议对比/TCP vs UDP]] 的 QUIC 支持 Connection ID 实现连接迁移，这跟 IPv6 不需要 NAT 有关——地址不再变化，连接更稳定

> **面试话术：** "IPv6 最大的变化是地址从 32 位扩展到 128 位，解决了地址耗尽问题。头部简化成固定 40 字节，路由处理更快。内置 IPSec 提升安全性，支持自动配置（SLAAC）。不过目前仍在过渡期，很多场景还是 IPv4 + NAT。"

## 七、面试回答模板

> **面试官：说一下 OSI 七层模型和 TCP/IP 四层模型的区别？**

**满分回答**：

"OSI 七层模型是理论上的标准参考模型，把网络通信分为物理层、数据链路层、网络层、传输层、会话层、表示层、应用层七层。它的设计非常严谨和理想化，但实际落地比较困难。

TCP/IP 四层模型是互联网实际使用的协议栈，分为应用层、传输层、网际层、网络接口层四层。它把 OSI 上三层的功能合并到了应用层，下两层合并到了网络接口层。

**面试时记住 TCP/IP 四层就行**，但要知道 OSI 的存在。实际工作中我们关注的是：应用层协议（HTTP/DNS）、传输层协议（TCP/UDP）、网络层（IP 路由）、底层硬件（网卡/交换机）。

打个比方：OSI 七层就像一部完美的宪法，TCP/IP 四层是实际施行的法律——宪法更完整但执行起来有成本，法律虽然简化但能落地。"

> **面试官：ARP 协议的作用？ARP 欺骗怎么防？**

"ARP 协议把 IP 地址翻译成 MAC 地址——因为在局域网内数据帧最终靠 MAC 地址投递，就像你知道对方门牌号但不知道具体住几楼几号，ARP 帮你查到精确位置。

ARP 欺骗是攻击者伪造 ARP 响应，把目标 IP 的 MAC 替换成攻击者的 MAC，导致流量被劫持。防御手段包括静态 ARP 绑定、交换机级 DAI 检测、端口安全限制，以及使用 HTTPS 加密确保即使流量被截获也无法解密。"

> **面试官：IP 地址怎么分类？子网掩码的作用？**

"IPv4 地址分 A/B/C/D/E 五类，A 类网络位 8 位可容纳 1600 万主机，C 类 24 位网络号只能 254 台。实际使用中通过子网掩码做更灵活划分——比如 `/24` 表示前 24 位网络号、后 8 位主机号，可用主机 254 台（去掉网络地址全 0 和广播地址全 1）。私有 IP（10.x、172.16~31.x、192.168.x）配合 NAT 使用，解决公网 IP 不够的问题。"

> **面试官：ping 用的什么协议？原理是什么？**

"ping 用 ICMP 协议，不是 TCP 也不是 UDP。发送 Echo Request 等对方回 Echo Reply 测 RTT。traceroute 也依赖 ICMP——通过逐跳递增 TTL 值，每个路由器 TTL 减到 0 时回 Time Exceeded 报文，从而探测完整路径。"
