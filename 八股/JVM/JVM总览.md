# JVM 虚拟机总览

> [!summary] 模块速记
> **JVM 四大核心：内存结构（堆栈方法区）、类加载（双亲委派）、GC（算法+收集器）、调优（参数+工具）。**
> **一条面试主线：new 一个对象 → 类加载 → 内存分配 → GC 回收 → OOM 排查。**

---

## 模块知识地图

```
JVM 虚拟机
  │
  ├── 01. 内存结构与对象创建
  │     ├── 5 大数据区：堆、栈、方法区、PC寄存器、本地方法栈
  │     ├── 堆分区：新生代(Eden/S0/S1) + 老年代
  │     ├── JDK8 Metaspace 替代永久代
  │     ├── 对象创建过程 + 内存分配（TLAB）
  │     └── 各区域 OOM 场景
  │
  ├── 02. 类加载机制
  │     ├── 类加载 7 个步骤
  │     ├── 双亲委派模型（为什么要打破？）
  │     ├── Tomcat/SPI 如何打破双亲委派
  │     └── Class.forName vs loadClass
  │
  ├── 03. GC 垃圾回收
  │     ├── 判断算法：引用计数、可达性分析（GC Roots）
  │     ├── 回收算法：标记清除、标记整理、复制、分代
  │     ├── 经典收集器：Serial/ParNew/Parallel/CMS/G1/ZGC
  │     └── CMS vs G1：三色标记、SATB、记忆集
```

## 一条主线串到底

```
new User()
  │
  ├── ① 类加载：User.class 怎么到 JVM？
  │     → 加载→验证→准备→解析→初始化
  │     → 双亲委派：AppCL→ExtCL→BootCL
  │
  ├── ② 内存分配：对象放在哪？
  │     → 栈上分配（逃逸分析）→ TLAB（线程本地分配缓冲）→ Eden区
  │
  ├── ③ 对象结构：对象头 + 实例数据 + 对齐填充
  │     → Mark Word 存锁信息、GC年龄、hashCode
  │
  ├── ④ GC 回收：对象什么时候被回收？
  │     → 可达性分析（GC Roots不可达）
  │     → MinorGC(Eden满了) / FullGC(老年代满了)
  │
  └── ⑤ OOM 排查：堆 dump → MAT 分析 → 找泄漏点
```
