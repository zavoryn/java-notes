# 深拷贝和浅拷贝

> [!summary] 我的速记
> **浅拷贝** = 复制对象自身，引用字段仍指向原对象（共享）；**深拷贝** = 连引用字段也递归复制出新对象。
> `clone()` 默认浅拷贝。深拷贝实现：① 手动递归 copy ② 序列化反序列化。不可变对象（如 String）浅拷贝安全。

## 一、核心概念

### 1.1 浅拷贝（Shallow Copy）

只复制对象本身，引用类型字段**只复制引用地址**，新旧对象共享内部的引用对象。

```text
原对象                   浅拷贝后
┌──────────┐            ┌──────────┐
│ name: ref├──→ "张三"  │ name: ref├──→ "张三"（同一个！）
│ age: 20  │            │ age: 20  │
│ list: ref├──→ [1,2,3] │ list: ref├──→ [1,2,3]（同一个！）
└──────────┘            └──────────┘
```

### 1.2 深拷贝（Deep Copy）

复制对象本身，**所有引用类型字段也递归复制**，新旧对象完全独立。

```text
原对象                   深拷贝后
┌──────────┐            ┌──────────┐
│ name: ref├──→ "张三"  │ name: ref├──→ "张三"（新复制）
│ age: 20  │            │ age: 20  │
│ list: ref├──→ [1,2,3] │ list: ref├──→ [1,2,3]（新复制）
└──────────┘            └──────────┘
```

## 二、Cloneable 接口与 clone()

### 2.1 基本用法（浅拷贝）

```java
public class Person implements Cloneable {
    String name;
    int age;
    Address address;    // 引用类型

    @Override
    public Person clone() {
        try {
            return (Person) super.clone();  // Object.clone() 是浅拷贝
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

> [!warning] 不实现 `Cloneable` 接口直接调用 `super.clone()` 会抛 `CloneNotSupportedException`。

### 2.2 浅拷贝的 Bug

```java
Person p1 = new Person("张三", 20, new Address("北京"));
Person p2 = p1.clone();

p2.address.city = "上海";     // 修改 p2 的地址
System.out.println(p1.address.city);  // "上海" ！！！ p1 的地址也被改了
// 原因：p1.address 和 p2.address 指向同一个 Address 对象
```

## 三、实现深拷贝的两种方式

### 方式 1：手动复制（推荐，性能好）

```java
public class Person implements Cloneable {
    String name;
    int age;
    Address address;
    List<String> hobbies;

    @Override
    public Person clone() {
        try {
            Person cloned = (Person) super.clone();  // 浅拷贝基本类型和 String
            cloned.address = this.address.clone();     // 深拷贝 Address
            cloned.hobbies = new ArrayList<>(this.hobbies); // 深拷贝 List
            return cloned;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

> [!tip] String 是不可变对象，浅拷贝也安全。但可变对象（List、Date、自定义类）必须手动深拷贝。

### 方式 2：序列化反序列化（通用但慢）

```java
public class DeepCopyUtil {
    @SuppressWarnings("unchecked")
    public static <T extends Serializable> T deepCopy(T obj) {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(obj);

            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (T) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}
```

> [!important] 序列化方式要求所有涉及的对象及其嵌套对象都实现 `Serializable` 接口。优势是无需手动递归，缺点是性能差。

## 四、拷贝构造函数 / 工厂方法

这是另一种实现深拷贝的方式，比 `clone()` 更直观：

```java
public class Person {
    String name;
    int age;
    Address address;

    // 拷贝构造函数
    public Person(Person other) {
        this.name = other.name;
        this.age = other.age;
        this.address = new Address(other.address);  // Address 也要有拷贝构造
    }

    // 静态工厂方法
    public static Person newInstance(Person other) {
        return new Person(other);
    }
}
```

### 五种方式对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| `clone()` 浅拷贝 | 简单 | 共享引用，容易出 BUG |
| `clone()` 手动深拷贝 | 性能好 | 字段多时代码繁琐 |
| 序列化 | 通用，自动递归 | 性能差，必须 Serializable |
| 拷贝构造函数 | 直观，类型安全 | 需手动实现 |
| 工厂方法 | 可缓存，灵活 | 比构造函数多一层调用 |

## 五、面试话术

> "浅拷贝只复制对象本身，内部引用类型字段还是指向原对象，这就是为什么浅拷贝出来的两个对象共用一个内部 List，改一个另一个也会变。
>
> 深拷贝会递归复制所有引用类型字段，新旧对象之间完全独立。实现深拷贝有两种主流方式：第一种是手动实现 clone 方法，逐层克隆每个引用字段；第二种是通过序列化反序列化，这个方式比较通用，但性能差一些，而且要求所有类都实现 Serializable。
>
> 实际开发中我更推荐拷贝构造函数的方式，比 clone 更直观，也没有 Cloneable 接口那些坑。另外要注意 String 等不可变对象浅拷贝就行了，不用深拷贝。
>
> 面试时经常问的一个点是：为什么 `Object.clone()` 是 protected 的？因为它不希望所有类都能直接调用，而是想让子类重写后暴露为 public。"
