# Java 注解（Annotation）

> [!summary] 我的速记
> **核心元注解：** `@Target`(放哪) + `@Retention`(留多久)。
> **RetentionPolicy 三种：** SOURCE(编译丢) / CLASS(留字节码) / RUNTIME(运行时保留，最常用)。
> **常见注解：** `@Override`(编译检查) / `@Deprecated`(标记废弃) / `@FunctionalInterface`(函数式接口检查)。

## 一、什么是注解

### 定义

注解是 JDK5 引入的**元数据**机制，用于在代码中添加额外信息，可以嵌入到类、方法、字段等元素上。注解本身不改变代码逻辑，但可以通过**反射**或**编译期处理器**来读取和处理。

> [!tip]
> 注解 = 贴在代码上的"标签"，运行时或编译时被工具扫描识别。

### 核心组成

1. **定义注解**（元注解修饰）
2. **使用注解**（贴在代码上）
3. **处理注解**（反射读取 / APT 编译时处理）

---

## 二、元注解

元注解是**注解的注解**，用来定义注解的行为。

| 元注解 | 作用 | 常用取值 |
|---|---|---|
| `@Target` | 指定注解可以用在哪里 | `TYPE`（类）、`FIELD`（字段）、`METHOD`（方法）、`PARAMETER`（参数）、`CONSTRUCTOR`（构造方法）... |
| `@Retention` | 指定注解保留到哪个阶段 | `SOURCE`（源码）、`CLASS`（字节码）、`RUNTIME`（运行时） |
| `@Documented` | 是否包含在 javadoc 中 | 有则生成文档 |
| `@Inherited` | 子类是否继承父类注解 | 有则子类自动继承 |

### @Retention 三种策略（面试重点）

| 策略 | 保留阶段 | 反射能否读取 | 典型应用 |
|---|---|---|---|
| `RetentionPolicy.SOURCE` | 源码阶段，编译时丢弃 | 否 | `@Override`、`@SuppressWarnings` |
| `RetentionPolicy.CLASS` | 编译到 class 文件，JVM 加载时丢弃 | 否 | Lombok 注解（`@Data`、`@Getter`） |
| `RetentionPolicy.RUNTIME` | 一直保留到运行期 | 是 | Spring 注解（`@Autowired`、`@Component`） |

> [!warning]
> 默认 `@Retention` 是 `CLASS` 级别。如果自定义注解需要在运行时通过反射读取，**必须**设为 `RUNTIME`。

## 三、常见内置注解

| 注解 | 作用 | Retention |
|---|---|---|
| `@Override` | 标记方法重写父类方法，编译器校验 | SOURCE |
| `@Deprecated` | 标记已过时，不推荐使用 | RUNTIME |
| `@SuppressWarnings("unchecked")` | 抑制编译器警告 | SOURCE |
| `@FunctionalInterface` | 标记函数式接口，编译器校验 | RUNTIME |
| `@SafeVarargs` | 抑制可变参数泛型警告 | RUNTIME |

## 四、自定义注解

### 基本语法

```java
@Target({ElementType.METHOD, ElementType.TYPE})   // 可用于方法和类
@Retention(RetentionPolicy.RUNTIME)                // 运行时保留
@Documented                                        // 生成 javadoc
public @interface MyAnnotation {
    // 成员用"方法签名"定义
    String value() default "";          // 有默认值
    int priority() default 1;
    String[] tags() default {};
}
```

### 使用注解

```java
@MyAnnotation(value = "test", priority = 5)
public class Demo {

    @MyAnnotation(value = "hello", tags = {"important", "core"})
    public void doSomething() {
        // ...
    }
}
```

> [!tip] 如果只有一个成员且名为 `value`，可以省略名称：`@MyAnnotation("hello")`

### 通过反射读取注解

```java
Class<Demo> clazz = Demo.class;

// 读取类上的注解
MyAnnotation annotation = clazz.getAnnotation(MyAnnotation.class);
System.out.println(annotation.value());      // "test"
System.out.println(annotation.priority());   // 5

// 读取方法上的注解
Method method = clazz.getMethod("doSomething");
MyAnnotation methodAnnotation = method.getAnnotation(MyAnnotation.class);
for (String tag : methodAnnotation.tags()) {
    System.out.println(tag);                 // "important", "core"
}
```

## 五、注解处理器（APT）

### 概念

APT（Annotation Processing Tool）是 Java 编译期的注解处理机制。在编译阶段扫描和处理注解，可以**生成新的源代码**，不会改变已有代码。

### 工作原理

```
源码编译 → APT 扫描注解 → 生成新源码/报错 → 新源码再次编译 → 合并成 class 文件
```

### 典型应用

- **Lombok**：通过 APT 在编译期自动生成 getter / setter / 构造方法等
- **MapStruct**：编译期生成对象映射代码
- **AutoService**：自动生成 SPI 配置文件

### 实现注解处理器

```java
@SupportedAnnotationTypes("com.example.MyAnnotation")  // 处理哪些注解
@SupportedSourceVersion(SourceVersion.RELEASE_8)
public class MyProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                           RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(MyAnnotation.class)) {
            // 读取注解信息，生成代码或校验
        }
        return true;  // true 表示已处理，不需要后续处理器再处理
    }
}
```

## 六、Spring 框架中注解的应用

| 注解 | 作用 |
|---|---|
| `@Component` / `@Service` / `@Repository` / `@Controller` | 标记为 Spring Bean，让 IOC 容器管理 |
| `@Autowired` | 依赖自动注入 |
| `@Value` | 读取配置文件属性值 |
| `@Transactional` | 声明式事务管理 |
| `@RequestMapping` / `@GetMapping` / `@PostMapping` | HTTP 请求映射 |
| `@Configuration` / `@Bean` | Java 配置类 |
| `@SpringBootApplication` | 组合注解（= @Configuration + @EnableAutoConfiguration + @ComponentScan） |
| `@Aspect` / `@Before` / `@After` | AOP 切面编程 |

### Spring 处理注解的方式

1. **IOC 容器启动时**，扫描指定包路径下的类
2. 通过**反射**读取 `@Component`、`@Autowired` 等运行时注解
3. 创建 Bean 实例、注入依赖、应用 AOP 代理
4. 核心依赖的就是注解的 `RUNTIME` 保留策略

## 七、面试话术

> Java 注解是 JDK5 引入的元数据标记机制，用于在代码中添加额外信息。注解本身不改变代码逻辑，需要通过反射或编译期处理器来读取和处理。
>
> 定义注解时用四大元注解：`@Target` 指定应用范围，`@Retention` 指定保留策略。`@Retention` 有三种级别：`SOURCE`（编译时丢弃，如 `@Override`）、`CLASS`（保留到字节码但运行时丢弃，如 Lombok 注解）、`RUNTIME`（保留到运行时，如 Spring 注解）。自定义注解如果要通过反射读取，必须设为 `RUNTIME`。
>
> 处理注解有两种方式：一是**运行时通过反射**读取（Spring IOC 的做法），二是**编译期通过 APT** 生成代码（Lombok 的做法）。Spring 框架大量使用了运行时注解来实现 IOC 和 AOP，而 Lombok 利用 APT 在编译期生成代码。两者配合让 Java 从繁重的 XML 配置走向了注解驱动开发。
