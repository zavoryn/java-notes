# Java 反射（Reflection）

> [!summary] 我的速记
> **反射 = 拿到类的"说明书"（Class 对象），看它有什么字段/方法/构造器，然后动态调用。**
>
> **记住 3 个面试点：**
> 1. 三种获取方式 → `类名.class` / `对象.getClass()` / `Class.forName()`
> 2. Declared = 本类所有含 private（不含继承）；非 Declared = public 含继承
> 3. `setAccessible(true)` = 破 private 门槛 + 提性能
>
> **用在哪？** Spring IOC 创建 Bean、依赖注入、JUnit 找 @Test、Jackson 序列化——全是反射。

## 一、什么是反射

### 核心定义

反射是 Java 提供的在**运行时**动态获取类的信息、操作类的属性和方法的能力。它让 Java 从"编译时强类型"变成"运行时也具备一定的动态能力"。

> [!tip] 一句话总结
> 反射就是在运行时拿到类的"说明书"（Class 对象），通过说明书看这个类有什么字段、什么方法、什么构造方法，然后动态调用。

### Class 对象是什么

JVM 在加载类时，会在**堆**中创建一个唯一的 `java.lang.Class` 对象，这个对象包含了该类的完整结构信息（字段、方法、构造方法、接口、注解等）。**反射的一切操作都基于这个 Class 对象**。

```java
// 一个类在 JVM 中只有一个 Class 对象（单例）
Class<String> c1 = String.class;
Class<String> c2 = String.class;
System.out.println(c1 == c2);  // true
```

## 二、获取 Class 对象的三种方式

```java
// 方式1：类名.class（最常用，编译期已知）
Class<Student> clazz1 = Student.class;

// 方式2：对象.getClass()（已有对象实例）
Student stu = new Student();
Class<? extends Student> clazz2 = stu.getClass();

// 方式3：Class.forName("全限定类名")（运行时动态加载，常用于配置文件）
Class<?> clazz3 = Class.forName("com.example.Student");
```

| 方式                | 适用场景      | 注意事项                  |
| ----------------- | --------- | --------------------- |
| `类名.class`        | 编译期知道类名   | 最安全，编译期检查             |
| `对象.getClass()`   | 已有对象实例    | 会触发父类静态初始化（如果有）       |
| `Class.forName()` | 运行期动态决定类名 | 会触发类的初始化（执行静态代码块），最灵活 |

> [!warning] `ClassLoader.loadClass()` 不会触发初始化，`Class.forName()` 会。Spring IOC 用 `Class.forName()` 就是因为要触发静态代码块的执行。

## 三、反射能做什么

### 1. 创建对象

```java
Class<Student> clazz = Student.class;

// 方式1：调用无参构造方法
Student s1 = clazz.getDeclaredConstructor().newInstance();

// 方式2：调用有参构造方法
Constructor<Student> constructor = clazz.getDeclaredConstructor(String.class, int.class);
Student s2 = constructor.newInstance("张三", 18);

// 方式3：Class.newInstance() 已废弃（JDK9+），不推荐
```

### 2. 获取 / 调用方法

```java
Class<Student> clazz = Student.class;
Object obj = clazz.getDeclaredConstructor().newInstance();

// getMethods(): 获取所有 public 方法（包括继承的）
Method[] publicMethods = clazz.getMethods();

// getDeclaredMethods(): 获取本类声明的所有方法（包括 private）
Method[] allMethods = clazz.getDeclaredMethods();

// 调用指定方法
Method setName = clazz.getDeclaredMethod("setName", String.class);
setName.invoke(obj, "张三");

// 调用私有方法，需要 setAccessible(true)
Method privateMethod = clazz.getDeclaredMethod("secret");
privateMethod.setAccessible(true);
privateMethod.invoke(obj);
```

### 3. 获取 / 修改字段

```java
Class<Student> clazz = Student.class;
Object obj = clazz.getDeclaredConstructor().newInstance();

// getFields(): 获取所有 public 字段（包括继承的）
Field[] publicFields = clazz.getFields();

// getDeclaredFields(): 获取本类声明的所有字段（包括 private）
Field[] allFields = clazz.getDeclaredFields();

// 修改私有字段
Field nameField = clazz.getDeclaredField("name");
nameField.setAccessible(true);  // 暴力破解，跳过权限检查
nameField.set(obj, "李四");

// 读取私有字段
String name = (String) nameField.get(obj);
```

### 4. 获取构造方法

```java
Class<Student> clazz = Student.class;

// getConstructors(): 获取所有 public 构造方法
Constructor<?>[] constructors = clazz.getConstructors();

// getDeclaredConstructors(): 获取所有构造方法（包括 private）
Constructor<?>[] allConstructors = clazz.getDeclaredConstructors();
```

## 四、关键 API 对比

> [!important] Declared vs 非 Declared

| API                         | 范围                             |
| --------------------------- | ------------------------------ |
| `getFields()`               | 所有 **public** 字段（包括父类继承的）      |
| `getDeclaredFields()`       | **本类声明**的所有字段（包括 private，不含继承） |
| `getMethods()`              | 所有 **public** 方法（包括父类继承的）      |
| `getDeclaredMethods()`      | **本类声明**的所有方法（包括 private，不含继承） |
| `getConstructors()`         | 所有 **public** 构造方法             |
| `getDeclaredConstructors()` | **本类**所有构造方法（包括 private）       |

### setAccessible(true) 的作用

- 关闭 Java 语言层面的**访问权限检查**
- 使反射可以访问 `private` 成员
- 同时也**提高反射调用的性能**（跳过权限检查）

## 五、反射的应用场景

> [!tip] 面试高频：反射在框架中无处不在

| 场景             | 说明                                               |
| -------------- | ------------------------------------------------ |
| **Spring IOC** | 读取 XML/注解配置，通过反射创建 Bean 实例、注入依赖                  |
| **JDBC**       | `Class.forName("com.mysql.cj.jdbc.Driver")` 加载驱动 |
| **动态代理**       | JDK 动态代理 `Proxy.newProxyInstance()` 依赖反射         |
| **注解处理器**      | 通过 `getAnnotation()` 读取注解信息                      |
| **序列化框架**      | Jackson/Gson 通过反射读写对象字段                          |
| **单元测试**       | JUnit 通过反射找到 `@Test` 方法并调用                       |
| **IDE 自动提示**   | 通过反射获取类的方法列表                                     |

### Spring IOC 简化示意

```java
// 模拟 Spring IOC 的反射流程
public class SimpleContainer {
    private Map<String, Object> beans = new HashMap<>();

    public void registerBean(String className) throws Exception {
        Class<?> clazz = Class.forName(className);           // 1. 加载类
        Object bean = clazz.getDeclaredConstructor().newInstance(); // 2. 反射创建对象

        // 3. 反射注入依赖（简化）
        for (Field field : clazz.getDeclaredFields()) {
            if (field.isAnnotationPresent(Autowired.class)) {
                field.setAccessible(true);
                field.set(bean, getBean(field.getType()));   // 注入依赖
            }
        }
        beans.put(className, bean);
    }
}
```

## 六、反射的性能问题及优化

### 性能开销来源

1. **类型检查开销**：反射调用绕过了编译期优化
2. **安全性检查**：每次调用都要检查访问权限
3. **装箱 / 拆箱**：`Method.invoke()` 的参数和返回值都是 Object
4. **Method 查找开销**

### 优化策略

```java
// 1. setAccessible(true)：关闭权限检查
Method method = clazz.getDeclaredMethod("getName");
method.setAccessible(true);  // 提升性能

// 2. 缓存 Class / Method / Field 对象（避免重复查找）
public class ReflectCache {
    private static final Map<String, Method> METHOD_CACHE = new ConcurrentHashMap<>();
    private static final Map<String, Field> FIELD_CACHE = new ConcurrentHashMap<>();

    public static Method getMethod(Class<?> clazz, String name, Class<?>... params) {
        // ... 从缓存中获取或加载
    }
}

// 3. 对高频调用，考虑使用 MethodHandle（JDK7+）或 LambdaMetafactory 代替
```

### 性能对比建议

| 调用方式                   | 性能排序 | 适用场景      |
| ---------------------- | ---- | --------- |
| 直接调用                   | 最快   | 已知类型      |
| MethodHandle           | 较快   | 替代反射的高频场景 |
| 反射（setAccessible=true） | 一般   | 中频动态调用    |
| 反射（默认）                 | 较慢   | 低频调用      |

## 原理深挖

### `Class.forName()` 和 `ClassLoader.loadClass()` 的根本区别

```java
// loadClass：只加载字节码，不执行 static{} 静态初始化块
ClassLoader.getSystemClassLoader().loadClass("com.mysql.cj.jdbc.Driver");

// forName：加载 + 初始化，执行 static{} 块
Class.forName("com.mysql.cj.jdbc.Driver");
```

MySQL 驱动在 static{} 里把自己注册到 DriverManager。如果只用 loadClass，驱动不会自注册，后续 getConnection 找不到驱动。这就是 JDBC 早期必须写 `Class.forName()` 的原因（JDBC4 后有了 SPI 自动发现，这行可以省略）。

### `setAccessible(true)` 为什么能破 private？

`Method.invoke()` 源码里有一个 `override` 字段，`setAccessible(true)` 就是把它设为 true。每次调用之前，JDK 源码执行 `Reflection.quickCheckMemberAccess()`——如果 `override == true`，直接跳过 Java 语言层面的访问检查。这不是 JVM 安全漏洞，是 JDK 有意留的"后门"，专供框架使用。JDK9+ 模块化系统增加了一层检查，但原理不变。

## 面试回答框架

> [!info] 问：讲讲反射？

**① 一句话定位：** 反射是 Java 在运行时动态获取类信息并操作类的能力，核心依赖 JVM 为每个类创建的 Class 对象。

**② 核心展开：**

获取 Class 有三种方式——类名.class 编译期确定、对象.getClass()、Class.forName() 运行时动态加载。特别注意 Class.forName() 会触发类初始化（执行 static 块），而 ClassLoader.loadClass() 不会——Spring IOC 正是利用这个特性触发 Bean 的静态逻辑。

反射四大核心操作：通过 Constructor 创建对象、Method.invoke() 调用方法、Field.set/get 操作字段、getAnnotation 读取注解信息。Declared 系列拿本类所有成员含 private（不含继承），非 Declared 只拿 public（含继承）。**setAccessible(true)** 两个作用：突破 private 限制 + 关闭安全检查提升性能。

**③ 框架落地：**
解决框架作者和业务代码的预见性，spring ioc（通过反射创建bean，读取autowird，实现依赖注入） 和mp 的ORM（通过查询的结果集合，通过反射动态赋值给pojo对象的私有属性）,
Spring IOC 的 Bean 实例化是 `Class.forName + newInstance`，依赖注入是 `Field.set`；动态代理的 `InvocationHandler.invoke()` 里用 `method.invoke(target, args)` 反射调用目标方法；Jackson 序列化靠反射读写字段；JUnit 靠反射找 @Test 方法执行。没有反射，这些框架都不存在。

> [!question] 追问：反射有什么缺点？

> 三个方面：一是性能差——绕过了 JIT 编译优化，每次调用做安全检查，参数和返回值都是 Object 要装箱拆箱；二是破坏封装——可以绕过 private 访问；三是失去编译期类型安全。优化方式：setAccessible(true)、缓存 Class/Method 对象、高频调用用 MethodHandle 替代。
> 1.性能：反射里面包含一系列的动态查找和验证，比直接调用慢
> 2.破环封装
> 3.类型不安全：编译时不坚持是否存在类，运行时候报错
