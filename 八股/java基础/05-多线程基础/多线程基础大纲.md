## 1. 线程的 6 种状态

```
NEW → RUNNABLE → BLOCKED / WAITING / TIMED_WAITING → TERMINATED
```

- **NEW**：线程创建但未启动（`new Thread()` 后、`start()` 前）
- **RUNNABLE**：可运行状态，包含 Ready（就绪）和 Running（运行中）
- **BLOCKED**：等待获取锁，如 `synchronized` 竞争失败时阻塞
- **WAITING**：无限期等待，如 `wait()`、`join()`
- **TIMED_WAITING**：限时等待，如 `sleep(ms)`、`wait(ms)`、`join(ms)`
- **TERMINATED**：线程执行完毕

## 3. synchronized 基本用法

- **对象锁**：锁住实例对象，作用于实例方法或 `synchronized(this)` 代码块
- **类锁**：锁住 Class 对象，作用于静态方法或 `synchronized(Xxx.class)` 代码块

```java
// 对象锁
public synchronized void method1() { }
// 等价于
public void method1() { synchronized(this) { } }

// 类锁
public static synchronized void method2() { }
// 等价于
public static void method2() { synchronized(Xxx.class) { } }
```

> 锁升级、Monitor 机制等内容详见 [[../../多线程/02-synchronized锁升级/synchronized锁升级]]

## 4. volatile 关键字

三大特性：
- **可见性**：一个线程修改 volatile 变量后，其他线程立即可见（强制刷主内存）
- **禁止指令重排**：通过内存屏障防止 JVM 指令重排序
- **不保证原子性**：`i++` 这种复合操作仍非原子，需要 `synchronized` 或 `AtomicInteger`

> JMM、内存屏障等内容详见 [[../../多线程/01-JMM与volatile/JMM与volatile]]

## 5. wait/notify/notifyAll 机制

- 三者都是 `Object` 类的方法，必须在 `synchronized` 块中调用
- `wait()`：释放锁并进入 WAITING 状态，等待被唤醒
- `notify()`：随机唤醒一个等待线程
- `notifyAll()`：唤醒所有等待线程

```java
synchronized (lock) {
    while (condition) {  // 用 while 而非 if，防止虚假唤醒
        lock.wait();
    }
    // 执行业务逻辑
    lock.notifyAll();
}
```

> 详见 [[../../多线程/多线程总览|多线程总览]]（wait/notify 深入见多线程模块）

## 6. sleep vs wait 区别

| 对比维度    | sleep                  | wait                             |
| ------- | ---------------------- | -------------------------------- |
| 所属类     | `Thread` 静态方法          | `Object` 实例方法                    |
| 锁释放     | **不释放锁**               | **释放锁**                          |
| 唤醒方式    | 时间到自动醒 / `interrupt()` | `notify()` / `notifyAll()` / 时间到 |
| 使用场景    | 暂停执行                   | 线程间通信（生产者-消费者）                   |
| 必须在同步块？ | 不需要                    | 必须在 `synchronized` 块内            |

> 口诀：**sleep 抱着锁睡觉，wait 放下锁等待。**
