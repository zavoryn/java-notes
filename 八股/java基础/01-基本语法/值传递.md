# 值传递

> [!summary] 我的速记
> **Java 只有值传递，没有引用传递。**
> - 基本类型：传值的副本，改副本不影响原值
> - 引用类型：传"引用地址"的副本，可通过副本改对象内容，但改不了引用指向
> - **经典面试题：swap 方法无法交换两个对象**——因为传的是引用副本，swap 里交换的是副本，不影响原变量。
> - **String 的"幻觉"**：String 不可变，所以传参修改操作实际是重新赋值引用副本，原引用不变。

## 一、核心结论

> [!important] **Java 只有值传递，没有引用传递。**

无论传的是基本类型还是引用类型，传递的都是**值的副本**：
- 基本类型 → 传递**值的副本**
- 引用类型 → 传递**引用地址的副本**

## 二、基本类型传值

```java
public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a);  // 10（没变！）
}

static void change(int x) {
    x = 20;   // x 是 a 的副本，修改 x 不影响 a
}
```

```text
调用前： main: [a = 10]
调用时： change: [x = 10]（a 的副本，独立）
x = 20 后：change: [x = 20]  main: [a = 10]（a 不受影响）
```

## 三、引用类型传值（最容易混淆）

```java
public static void main(String[] args) {
    User user = new User("张三");
    change(user);
    System.out.println(user.name);  // "李四"（对象内容被改了！）
}

static void change(User u) {
    u.name = "李四";    // 通过副本引用修改了对象内容
    u = new User("王五"); // 副本引用指向新对象，不影响原引用
}
```

```text
调用前：   main: [user → User("张三")]
调用时：   change: [u → User("张三")]（复制了引用地址）
u.name="李四" 后： 两个引用都指向被修改后的对象
u=new User("王五") 后： mains的user 还是指向 User("李四")
```

> [!tip] 通过方法参数**可以修改对象的内部状态**（通过副本指向同一对象），但**不能改变原引用指向新的对象**。

## 四、经典 Swap 问题

```java
// 交换两个 int — 不可能
public static void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}
// main: swap(x, y) 之后 x y 不变，因为传的是副本

// 交换两个对象 — 也不可能
public static void swap(User a, User b) {
    User temp = a;
    a = b;
    b = temp;
}
// main: swap(u1, u2) 之后 u1 u2 不交换
// 因为传的是引用副本，交换的是副本的指向，不影响原引用
```

> [!abstract] Java 中**无法通过方法交换两个变量**。无论基本类型还是引用类型。

### 变通方案

```java
// 1. 用数组或容器包一层
public static void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}

// 2. 用包装类（AtomicReference 等）
public static void swap(AtomicReference<User> a, AtomicReference<User> b) {
    User temp = a.get();
    a.set(b.get());
    b.set(temp);
}
```

## 五、String 的"幻觉"

```java
public static void main(String[] args) {
    String s = "hello";
    changeStr(s);
    System.out.println(s);  // "hello"（没变！）
}

static void changeStr(String str) {
    str = "world";   // str 指向新对象，s 不变
}
// 原因：String 不可变 + 值传递 = 无法修改原引用
```

> [!tip] 容易误以为 String 被改了，其实是因为 String 不可变，`str = "world"` 是让形参指向了新对象，而不是修改原对象。

## 六、与 C++ 的对比（帮助理解）

```text
C++ 三种传递方式：
  - 值传递：void foo(int x)       → Java 只有这种
  - 引用传递：void foo(int &x)    → Java 没有
  - 指针传递：void foo(int *x)    → Java 没有

很多人把"传递引用"和"引用传递"搞混：
  - 传递引用 = 传的是对象的引用地址（指针）
  - 引用传递 = 形参是实参的别名，改形参=改实参（Java 没有）
```

## 原理深挖

### "Java 只有值传递" 的底层证据

```java
public static void swap(Object a, Object b) {
    Object temp = a;
    a = b;
    b = temp;
}
String x = "A", y = "B";
swap(x, y);
// x 还是 "A"，y 还是 "B"——没有交换！
```

如果 Java 真的是"引用传递"，形参 a/b 应该直接就是 x/y 本身，修改 a/b 会直接影响 x/y。但实际行为是：**a 拿到的是 x 的值的副本（即 x 所存的对象地址的副本）**，a 和 x 是两个独立的变量，a 重新指向新对象不影响 x。这就是值传递的定义——传递的是值的拷贝。

### String "幻觉" 的真相

```java
void change(String s) { s = "world"; }
String x = "hello";
change(x);
// x 还是 "hello"！很多人以为 "引用类型，传了引用，改了应该生效"
```

这不是因为 String 特殊，而是同样的值传递规则：s 拿到 x 地址的副本。`s = "world"` 让副本 s 指向新对象——原引用 x 不动。再加上 String 不可变（不能通过 `s.setCharAt(0, 'x')` 修改内容），造成了"传引用也没用"的假象。

## 面试回答框架

> [!info] 问：Java 是值传递还是引用传递？

**① 一句话定论：** Java 只有值传递，没有引用传递。

**② 核心论证：** 基本类型传的是值的副本——改形参不影响实参。引用类型传的也是值的副本——不过这个"值"是对象的地址。副本也指向同一对象，所以可以通过副本修改对象内部状态，但副本本身重新赋值不影响原引用。这是"值传递"的正确定义——传递变量值的拷贝。

**③ 经典反证：** swap 方法无法交换两个对象——如果 Java 是引用传递，swap 里交换形参应该直接影响实参，但实际不会。这道题本身就是判断标准。

> [!question] 追问：基本类型值传递，引用类型引用传递——这个说法错在哪？

> 错在把"传递的值是引用/地址"和"引用传递"概念混淆了。引用传递的特点是形参和实参是同一个变量，修改形参直接影响实参。Java 中引用类型的形参和实参是两个独立变量，只是他们的值恰好都指向同一对象。区别在于：值传递时形参重新赋值不影响实参；引用传递则会。Java 做不到。
