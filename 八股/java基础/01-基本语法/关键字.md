# 关键字：static / final / this / super

> [!summary] 我的速记
> **static** = 类级别，所有实例共享，无 `this`，不能调非 static。
> **final** = 不可变（类→不可继承，方法→不可重写，变量→不可修改；引用类型内容可变）。
> **this** = 当前对象，`this()` 调本类构造（必须第一行）。
> **super** = 父类引用，`super()` 调父类构造（必须第一行），`this()` 和 `super()` 不能同时出现。

## 一、static（静态）

### 1.1 核心概念

`static` 修饰的成员属于**类级别**，不属于某个实例。所有实例共享同一个静态变量。

```java
public class Student {
    static String school = "清华大学";   // 所有 Student 共享
    String name;                        // 每个实例独立

    static void printSchool() {         // 静态方法
        System.out.println(school);
    }
}
```

### 1.2 静态变量 vs 实例变量

| 对比维度 | 静态变量 | 实例变量 |
|---------|--------|---------|
| 归属 | 类（Class） | 每个对象（Instance） |
| 内存 | 方法区/元空间 | 堆内存 |
| 加载时机 | 类加载时初始化 | 对象创建时初始化 |
| 访问方式 | `类名.变量` / `对象.变量` | 只能 `对象.变量` |
| 生命周期 | 与类相同（类卸载才回收） | 与对象相同（GC 回收） |

### 1.3 静态代码块 vs 普通代码块 vs 构造方法

```java
public class Order {
    static { System.out.println("1. 静态代码块"); }   // 类加载时执行一次
    { System.out.println("2. 普通代码块"); }          // 每次 new 都执行
    public Order() { System.out.println("3. 构造方法"); }

    public static void main(String[] args) {
        new Order(); // 输出：1 → 2 → 3
        new Order(); // 输出：2 → 3（静态代码块不再执行）
    }
}
```

> [!important] 执行顺序：**静态代码块 → 普通代码块 → 构造方法**（静态只执行一次）

### 1.4 静态内部类

```java
public class Outer {
    static class StaticInner {   // 不持有外部类引用
        void say() { System.out.println("静态内部类"); }
    }

    class Inner {                // 持有外部类引用（Outer.this）
        void say() { System.out.println("成员内部类"); }
    }
}

// 实例化
Outer.StaticInner si = new Outer.StaticInner();          // 不需要外部类实例
Outer.Inner in = new Outer().new Inner();                 // 必须先有外部类实例
```

### 1.5 为什么 static 方法不能调用非 static 成员？

```java
public class Test {
    int num = 10;
    static void method() {
        // System.out.println(num);  // 编译错误！
        // this.num;                 // 编译错误！
    }
}
```

> [!abstract] 静态方法属于类，调用时可能没有任何对象存在。非静态成员依赖具体对象，没有对象就无法访问。就像"班级名单"不能访问"某个学生的铅笔"，因为学生可能还没入学。

### 1.6 static final 常量

```java
public static final int MAX_SIZE = 100;    // 全局常量
public static final String APP_NAME = "MyApp";  // 编译期常量
```

`static final` 组合表示**全局常量**，编译期就能确定值，JVM 会做内联优化。

## 二、final（不可变）

### 2.1 三种用法

| 应用位置 | 作用 | 示例 |
|---------|------|------|
| 修饰类 | 类不可被继承 | `final class String {}` |
| 修饰方法 | 方法不可被子类重写 | `final void init() {}` |
| 修饰变量 | 变量值不可修改 | `final int x = 10;` |

### 2.2 final 修饰引用类型

```java
final StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");    // OK——修改的是对象内容
// sb = new StringBuilder();  // 编译错误——不能改变引用指向
```

> [!warning] final 修饰引用类型时，**引用地址不可变，但对象内容可以变**。

### 2.3 final 变量的赋值时机

```java
public class Demo {
    final int a = 10;                     // 1. 定义时赋值
    final int b;
    { b = 20; }                          // 2. 代码块中赋值
    final int c;
    public Demo() { c = 30; }            // 3. 构造方法中赋值
    // 三种方式必选其一，且只能赋值一次
}
```

## 三、this

### 3.1 核心含义

`this` 是**当前对象的引用**，指向调用方法的那个实例。

```java
public class Person {
    String name;

    public Person(String name) {
        this.name = name;   // this.name 是实例变量，name 是参数
    }

    public void print() {
        System.out.println(this.name);   // this 可省略
    }
}
```

### 3.2 构造函数中 this()

```java
public class Person {
    String name;
    int age;

    public Person() {
        this("无名氏", 0);    // 调用另一个构造函数，必须在第一行
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

> [!important] `this()` 调用同类其他构造方法，**必须放在第一行**。

## 四、super

### 4.1 核心含义

`super` 是**父类对象的引用**，用于访问父类的成员。

```java
class Father {
    String name = "Father";
    void say() { System.out.println("Father say"); }
}

class Son extends Father {
    String name = "Son";

    void test() {
        System.out.println(name);          // Son（本类）
        System.out.println(super.name);    // Father（父类）
        System.out.println(this.name);     // Son（等同 name）
        super.say();                       // 调用父类方法
    }
}
```

### 4.2 super() 调用父类构造

```java
class Father {
    Father(String name) { System.out.println("Father: " + name); }
}

class Son extends Father {
    Son() {
        super("张三");    // 调用父类有参构造，必须在第一行
        System.out.println("Son 构造");
    }
}
```

> [!important] `super()` 和 `this()` **不能同时出现**在同一个构造函数中，因为它们都要求放在第一行。如果都不写，编译器默认加 `super()`（无参）。

### 4.3 this vs super 对比

| 比较维度 | this | super |
|---------|------|-------|
| 指向 | 当前对象 | 父类对象的引用 |
| 访问本类成员 | 可以 | 不可以（被覆盖的除外） |
| 访问父类成员 | 要显式覆盖才能访问 | 直接访问 |
| 调用构造 | `this()` 同类构造 | `super()` 父类构造 |
| 构造函数位置 | 必须第一行 | 必须第一行 |

## 五、面试话术

> "static 关 键字表示类级别的成员，属于类而不是实例，所以在类加载时初始化。静态方法中不能直接调用非静态成员，因为调用静态方法时可能还没有创建任何对象。
>
> final 关键字表示"不可改变"：修饰类不能被继承，修饰方法不能被重写，修饰变量必须且只能赋值一次。特别要注意 final 修饰引用类型时，引用地址不能改，但对象的内容可以变。
>
> this 代表当前对象的引用，常用来区分成员变量和局部变量。super 代表父类引用，子类构造方法中默认会调用 super()，如果父类没有无参构造，子类必须显式调用 super(参数)。
>
> 这两个调用——super() 和 this()——都必须放在构造函数的第一行，因此不能同时出现。"
