# 数据类型

## 一、8 种基本数据类型

| 类型 | 包装类 | 内存占用 | 取值范围 | 默认值 |
|------|--------|---------|---------|--------|
| byte | Byte | 1 字节 | -128 ~ 127 | 0 |
| short | Short | 2 字节 | -2^15^ ~ 2^15^-1（约 -3.2 万 ~ 3.2 万） | 0 |
| int | Integer | 4 字节 | -2^31^ ~ 2^31^-1（约 -21 亿 ~ 21 亿） | 0 |
| long | Long | 8 字节 | -2^63^ ~ 2^63^-1 | 0L |
| float | Float | 4 字节 | ~ ±3.4e38，精度 6-7 位 | 0.0f |
| double | Double | 8 字节 | ~ ±1.8e308，精度 15-16 位 | 0.0d |
| char | Character | 2 字节 | 0 ~ 65535（Unicode） | '\u0000' |
| boolean | Boolean | JVM 依赖 | true / false | false |

> [!note] boolean 大小：JVM 规范未明确定义，HotSpot 中 boolean 数组用 byte 实现（1 字节），单独 boolean 字段可能用 int（4 字节）。

```java
long l = 10000000000L;    // 必须加 L/l，否则默认 int 编译报错
float f = 3.14f;           // 必须加 F/f，否则默认 double 编译报错
```

## 二、引用类型

除了 8 种基本类型，其他全是引用类型：
- **类**（Class）：String、Object、自定义类等
- **接口**（Interface）：Runnable、Comparable 等
- **数组**（Array）：`int[]`、`String[]` 等
- **枚举**（Enum）
- **注解**（Annotation）

```java
String s = "hello";      // 引用类型，s 存的是地址
int[] arr = {1, 2, 3};   // 数组也是对象，arr 存的是地址
```

> [!important] 基本类型存的是**值本身**（栈中），引用类型存的是**对象的地址**（栈中存引用，堆中存对象）。

## 三、自动类型提升

### 3.1 自动提升规则

```text
byte → short → int → long → float → double
        char  ↗
```

> [!tip] 两个操作数运算时，先将小范围的类型**自动提升**为大范围的类型，再计算结果。

```java
byte b1 = 10, b2 = 20;
// byte b3 = b1 + b2;          // 编译错误！b1 + b2 自动提升为 int
int b3 = b1 + b2;              // OK，结果为 int

long l = 100L;
float f = 3.14f;
float result = l + f;          // long + float → float（因为 float 范围更大）
```

> [!warning] **byte、short、char 参与运算时，自动提升为 int！**

### 3.2 常见陷阱

```java
short s1 = 10;
short s2 = 20;
short s3 = s1 + s2;    // 编译错误！s1 + s2 提升为 int，不能赋给 short
short s4 = (short)(s1 + s2);  // OK，强制转换

char c = 'a';
int i = c + 1;         // OK，char 自动提升为 int，i = 98 ('b')
```

## 四、强制类型转换

### 4.1 语法

```java
double d = 3.14;
int i = (int) d;       // 强制转换，i = 3（丢小数部分）

long l = 3000000000L;
int n = (int) l;       // 溢出！n = -1294967296
```

> [!danger] 强制类型转换可能导致精度丢失或数据溢出，需谨慎使用。

### 4.2 精度丢失示例

```java
// 小数降级
float f = (float) 3.14159265358979;  // 只保留 6-7 位精度

// 大范围降级（溢出）
int huge = (int) 2147483648L;        // 溢出为 -2147483648

// char 与 int 互转
int code = 65;
char ch = (char) code;               // ch = 'A'
```

## 五、byte/short/char 运算的特殊规则

```java
byte a = 1;
byte b = 2;
byte c = (byte)(a + b);   // 必须强转，因为 a + b 自动提升为 int

// 复合赋值运算符自带强转
byte d = 1;
d += 2;                   // OK，等价于 d = (byte)(d + 2)
d++;                      // OK，等价于 d = (byte)(d + 1)
```

> [!tip] `+=`、`++`、`--` 等复合赋值运算符**内置了隐式强制转换**，不会报编译错误。

## 六、switch 支持的类型

| JDK 版本 | 支持的类型 |
|---------|----------|
| JDK 1.5 以前 | byte、short、int、char |
| JDK 5（enum 引入） | + enum |
| JDK 7 | + String |
| JDK 14+（预览） | 模式匹配 + 多值 case（`case 1, 2, 3 ->`） |

> [!abstract] 不支持的类型：**long、float、double、boolean**。因为 switch 本质是跳转表（整数索引），浮点数精度和范围导致无法精确匹配。

```java
String fruit = "apple";
switch (fruit) {
    case "apple":  System.out.println("苹果"); break;
    case "banana": System.out.println("香蕉"); break;
    default:       System.out.println("未知");
    // 底层：先 hash 哈希值，再用 .equals() 比对，防止冲突
}
```

## 七、面试话术

> "Java 有 8 种基本类型，分成四类：整数型 byte/short/int/long，浮点型 float/double，字符型 char，布尔型 boolean。它们的内存占用从小到大的顺序是 byte 1 字节、short 2 字节、int 4 字节、long 8 字节、float 4 字节、double 8 字节、char 2 字节、boolean JVM 依赖。
>
> 自动类型提升遵循从小到大的原则：byte → short → int → long → float → double，char 也可以提升到 int。特别要注意的是 byte、short、char 三者参与运算时，会先自动提升为 int，所以结果必须用 int 接收或强制转回去。
>
> 强制类型转换可能造成两个问题：精度丢失（如 double 转 int 丢掉小数）和数据溢出（如 long 转 int 超范围后值变负数）。
>
> switch 语句支持 byte、short、int、char 以及它们的包装类，JDK5 开始支持枚举，JDK7 开始支持 String。底层实现是用哈希值做索引再 equals 比对。"
