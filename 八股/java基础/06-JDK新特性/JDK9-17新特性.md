# JDK 9-17 新特性

> JDK 8 之后，Oracle 改为每 6 个月发布一个版本。其中 **JDK 11** 和 **JDK 17** 是 LTS（长期支持）版本。

---

## JDK 9（2017.9）

### 1. 模块化系统（JPMS, Java Platform Module System）

```java
// module-info.java
module com.example.mymodule {
    requires java.sql;           // 声明依赖
    exports com.example.api;    // 暴露包给其他模块
}
```

> 核心目标：更好的封装、显式依赖管理、裁剪 JRE 体积。实际企业开发中使用较少，Spring 等框架仍运行在 classpath 模式。

### 2. 集合工厂方法

```java
List<String> list = List.of("a", "b", "c");       // 不可变集合
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("k1", 1, "k2", 2);
```

> `List.of()` 返回**不可变集合**，不能 add/remove，不允许 null 元素。

### 3. 接口 private 方法

```java
public interface Calculator {
    default int sum(int a, int b) { return logAndCompute(a, b, a + b); }
    default int diff(int a, int b) { return logAndCompute(a, b, a - b); }

    // private 方法：用于复用 default 方法中的公共逻辑
    private int logAndCompute(int a, int b, int result) {
        System.out.printf("计算 %d 和 %d 的结果: %d%n", a, b, result);
        return result;
    }
}
```

### 4. 其他

- `try-with-resources` 增强：可以使用已存在的有效变量
- `Optional.ifPresentOrElse()`、`Optional.or()`
- `Stream.takeWhile()` / `Stream.dropWhile()`

## JDK 10（2018.3）

### `var` 局部变量类型推断

```java
var list = new ArrayList<String>();   // 编译器推断为 ArrayList<String>
var map = Map.of("k", 1);            // Map<String, Integer>
var stream = list.stream();

// 限制：
// ❌ 不能用于字段、方法参数、方法返回值
// ❌ 不能用于 Lambda 表达式
// ❌ 不能初始化为 null
// ✅ 只能用于局部变量
```

> 本质是**编译器语法糖**，编译后类型固定，不影响运行时。

## JDK 11（2018.9，LTS）

### 1. HTTP Client API（标准化）

```java
HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://example.com"))
    .GET()
    .build();
HttpResponse<String> response = client.send(request,
    HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
```

> 支持 HTTP/2、WebSocket、同步/异步调用。取代了老旧的 `HttpURLConnection`。

### 2. String 新增方法

```java
"  hello  ".strip();            // 去除首尾空白（比 trim() 更全，支持 Unicode 空白）
"\nline1\nline2".lines();       // 返回 Stream<String>，按行分割
"".isBlank();                   // 是否为空白字符串（空格、tab 等）
"hello".repeat(3);              // "hellohellohello"
```

### 3. 其他

- **ZGC**（实验性）：可扩展的低延迟垃圾回收器
- 可以直接运行单个 `.java` 源文件：`java Hello.java`

## JDK 14（2020.3）

### 1. Record 类型（预览，JDK 16 正式）

```java
// 一步到位定义不可变数据载体
public record Point(int x, int y) { }

// 自动生成：构造器、getter（无 get 前缀，直接 x()/y()）,
//           equals()、hashCode()、toString()
Point p = new Point(3, 5);
System.out.println(p.x());  // 3
```

> `record` 是**不可变**数据载体，所有字段隐式 `final`，适合 DTO/VO 等纯数据类，替代 Lombok 的 `@Data`。

### 2. `instanceof` 模式匹配（预览，JDK 16 正式）

```java
// 旧写法
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

// 新写法：一步到位
if (obj instanceof String s) {
    System.out.println(s.length());
}
```

## JDK 15（2020.9）

### 文本块（Text Blocks）

```java
// 旧写法
String json = "{\n" +
    "  \"name\": \"Alice\",\n" +
    "  \"age\": 25\n" +
    "}";

// 文本块：三个双引号，保留换行和缩进
String json = """
    {
      "name": "Alice",
      "age": 25
    }
    """;

// 去掉换行符（行尾加反斜杠）
String query = """
    SELECT id, name, age \
    FROM user \
    WHERE age > 18 \
    """;
// 结果: "SELECT id, name, age FROM user WHERE age > 18"
```

## JDK 17（2021.9，LTS）

### 1. Sealed Class（密封类）

```java
// 限制哪些类可以继承本类
public sealed class Shape
    permits Circle, Rectangle, Triangle { }

final class Circle extends Shape { }
final class Rectangle extends Shape { }
non-sealed class Triangle extends Shape { }  // non-sealed 允许进一步继承
```

> 密封类精确控制继承层次，结合 `switch` 模式匹配可以做到**穷尽性检查**。

### 2. Pattern Matching for switch（预览）

```java
// 功能更强大的 switch
Object obj = "hello";
String result = switch (obj) {
    case Integer i    -> "整数: " + i;
    case String s     -> "字符串长度: " + s.length();
    case null         -> "null 值";
    default           -> "unknown";
};
```

### 3. 其他

- **ZGC** 正式投产
- 移除 `Applet API`、`RMI Activation`
- 默认强封装 JDK 内部 API

## 面试话术

> "在 JDK 版本问题上，面试官主要关心的是你对版本演进的了解和实际使用经验。我会重点提到 JDK 8 是里程碑版本，引入了 Lambda 和 Stream。JDK 11 和 17 是后续的 LTS 版本，企业生产中 JDK 17 是当前推荐的主流版本——它带来了密封类、record 正式特性、ZGC 正式化等。record 和密封类和模式匹配 `switch` 是目前面试常问的新语法点。另外我实际工作中常用 `var` 做局部变量类型推断、文本块处理 JSON/SQL、`List.of()` 创建不可变集合。"

> "面试中如果问到为什么选 JDK 17 而不是 JDK 8，可以回答：JDK 17 是 LTS 长期支持版本，有更好的 GC 性能（ZGC/Shenandoah）、更现代的语言特性（record/sealed class/文本块/模式匹配）、更强的安全性（强封装内部 API），且 Spring Boot 3.x 和 Spring Framework 6 要求 Java 17 作为最低版本。"
