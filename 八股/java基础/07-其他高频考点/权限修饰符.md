# Java 权限修饰符

> [!summary] 我的速记
> **四级权限（窄→宽）：** private（仅本类）→ default（同包）→ protected（同包+跨包子类）→ public（全开放）。
> **关键规则：** 顶层类只 public/default；接口方法默认 `public abstract`，字段默认 `public static final`。
> **protected 陷阱：** 子类不能通过父类引用调 protected 方法（只能通过自己的继承链访问）。

## 1. 四种权限修饰符

| 修饰符 | 同类 | 同包子类 | 同包非子类 | 不同包子类 | 不同包非子类 |
|--------|:----:|:--------:|:----------:|:----------:|:------------:|
| `private` | ✅ | ❌ | ❌ | ❌ | ❌ |
| `default`（无修饰符） | ✅ | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ | ✅ |

## 2. 重要规则

### 2.1 类修饰符

```java
// ✅ 合法：顶层类只能用 public 或 default
public class PublicClass { }
class DefaultClass { }

// ❌ 不合法：顶层类不能是 private 或 protected
// private class PrivateClass { }        // 编译错误
// protected class ProtectedClass { }    // 编译错误

// ✅ 内部类可以使用所有四种修饰符
public class Outer {
    private class PrivateInner { }
    protected class ProtectedInner { }
    class DefaultInner { }
    public class PublicInner { }
}
```

### 2.2 接口方法/字段默认修饰符

```java
public interface MyInterface {
    // 接口方法默认是 public abstract
    void method();  // 等价于 public abstract void method();

    // 接口字段默认是 public static final（常量）
    int VALUE = 10;  // 等价于 public static final int VALUE = 10;

    // JDK 8: default 方法默认是 public
    default void doSomething() { }  // public 可省略

    // JDK 8: static 方法默认是 public
    static void util() { }  // public 可省略

    // JDK 9: private 方法（只能用于接口内部的 default/static 方法）
    private void helper() { }
}
```

### 2.3 protected 的访问规则

```java
package com.example.a;

public class Parent {
    protected void display() { }
}

package com.example.b;
import com.example.a.Parent;

public class Child extends Parent {
    public void test() {
        // ✅ 子类可以访问继承的 protected 方法
        super.display();
        this.display();

        // ❌ 不能通过父类引用访问 protected 方法
        Parent p = new Parent();
        // p.display();  // 编译错误！（必须在自己继承链中访问）

        Child c = new Child();
        c.display();  // ✅ 可以通过子类引用访问
    }
}
```

> [!important] protected 的关键规则
> `protected` 方法的**子类访问**仅限于**当前子类实例的继承链**中。不能在子类中通过父类引用直接调用另一个对象的 `protected` 方法。这个考察点非常高频！

## 3. 面试话术

> "Java 的四种访问权限从窄到宽依次是 private、default、protected、public。private 仅本类可见，通常用于封装内部状态。default 是包级访问权限，同一个包下的类可以互相访问，这也是 JDK 内部包的常见隔离手段。protected 比 default 多了一个跨包子类访问的权限，但有规则限制——子类只能通过自己的继承链访问 protected 成员，不能通过父类引用直接调用。public 完全开放。"

> "类修饰符方面，顶层类只能是 public 或 default，不能是 private 或 protected——因为 private 类没有意义（无法被使用），protected 类的访问范围也不比 default 大。但是内部类可以使用全部四种修饰符。接口中的方法默认是 public abstract，字段默认是 public static final，这些默认修饰符会在编译时自动加上。"

> "实际开发中，我遵循最小权限原则：成员变量默认 private 配合 getter/setter；只有需要被子类重写的方法才用 protected；对外 API 类用 public；工具类用 public 类名 + private 构造器防止实例化。"

## 4. 记忆口诀

> **private 自己知，default 同包知，protected 同包子包知，public 全天下知。**

> **顶层类只能 public 或 default，内部类四个全都有；接口方法全是 public，字段全是 static final 常量。**
