# Java 语法糖

> [!summary] 我的速记
> **语法糖** = 编译器层面的简化，编译后还原为基础语法，不影响字节码。
> **常见 8 种：** 自动装箱拆箱→valueOf/intValue；泛型→类型擦除；增强 for→迭代器；switch String→hashCode+equals；可变参数→数组；try-with-resources→finally close；Lambda→invokedynamic。
> **面试陷阱：** 自动装箱时 Integer -128~127 走缓存；Lambda 的 `this` 指向外部类（匿名内部类指向自身）。

## 1. 什么是语法糖？

> **语法糖（Syntactic Sugar）** 是编译器层面的**代码简化技巧**，让代码写起来更简洁、可读性更好，但**不影响 JVM 字节码层面的执行逻辑**。

核心要点：语法糖在**编译阶段**被编译器转换为对应的底层实现，编译后看字节码和手写底层代码是一样的。

## 2. 常见 Java 语法糖

### 2.1 自动装箱/拆箱

```java
// 源代码
Integer num = 100;      // 自动装箱
int val = num;          // 自动拆箱

// 编译后等价于
Integer num = Integer.valueOf(100);
int val = num.intValue();
```

> 注意：`-128 ~ 127` 范围内 `Integer.valueOf()` 会走缓存池，返回同一对象。这是面试高频陷阱题。

### 2.2 泛型擦除

```java
// 源代码
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);

// 编译后泛型被擦除
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 编译器自动插入类型转换
```

> Java 泛型是**伪泛型**，编译后类型信息被擦除，字节码层面所有泛型都是 `Object`（或上界类型）。

### 2.3 增强 for 循环（for-each）

```java
// 源代码
for (String s : list) {
    System.out.println(s);
}

// 编译后（实现 Iterable 接口的）
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}

// 编译后（数组形式）
String[] arr = list;
for (int i = 0; i < arr.length; i++) {
    String s = arr[i];
    System.out.println(s);
}
```

### 2.4 switch 支持 String

```java
// 源代码
switch (str) {
    case "A": ...; break;
    case "B": ...; break;
}

// 编译后等价于
String s = str;
int hashCode = s.hashCode();  // 先 switch hashCode
switch (hashCode) {
    case 65:  // "A".hashCode()
        if (s.equals("A")) { ... }  // 再用 equals 防冲突
        break;
    case 66:  // "B".hashCode()
        if (s.equals("B")) { ... }
        break;
}
```

> `switch` 实际上只支持 `int`（以及 char/byte/short 自动提升为 int），`String` 和 `enum` 都是语法糖。

### 2.5 可变参数

```java
// 源代码
public void print(String... args) {
    for (String s : args) {
        System.out.println(s);
    }
}
print("a", "b", "c");

// 编译后等价于
public void print(String[] args) { ... }
print(new String[]{"a", "b", "c"});
```

### 2.6 try-with-resources

```java
// 源代码（JDK 7+）
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    return br.readLine();
}

// 编译后等价于完整的 try-catch-finally
BufferedReader br = new BufferedReader(new FileReader("file.txt"));
Throwable var2 = null;
try {
    return br.readLine();
} catch (Throwable var12) {
    var2 = var12;
    throw var12;
} finally {
    if (br != null) {
        if (var2 != null) {
            try { br.close(); }
            catch (Throwable var11) { var2.addSuppressed(var11); }
        } else {
            br.close();
        }
    }
}
```

> `try-with-resources` 自动调用 `close()`，要求资源实现 `AutoCloseable` 接口。多个异常时用 `addSuppressed` 保存被压制的异常。

### 2.7 Lambda 表达式

```java
// 源代码
Runnable r = () -> System.out.println("hello");

// 编译后（简化示意）
// 使用 invokedynamic 指令 + LambdaMetafactory 在运行时生成函数式接口的实现
// 不会生成匿名内部类的 .class 文件
```

> Lambda 的实现不是匿名内部类（不会生成 `Xxx$1.class`），而是通过 `invokedynamic` 指令 + `LambdaMetafactory` 在运行时动态绑定，**不生成额外类文件，性能更好，没有 `this` 引用问题**。

### 2.8 其他语法糖

| 语法糖 | 编译后 |
|--------|--------|
| `==` 对包装类 | 装箱/拆箱操作 |
| 数字下划线 `1_000_000` | 直接去掉下划线 |
| 菱形运算符 `<>` | 编译器自动推断类型参数 |
| 条件编译 `if (false)` | 编译期直接优化掉（JIT 不编译死代码） |
| enum 枚举类 | 编译为继承 `java.lang.Enum` 的 final 类 |
| 内部类 | 编译为独立的 `.class` 文件 |

## 3. 面试话术

> "语法糖是编译器层面做的代码简化，它不改变底层字节码逻辑，只是让开发者写代码更简洁。常见的有：自动装箱拆箱本质是 `valueOf()` 和 `intValue()` 调用；泛型是伪泛型，编译后擦除类型信息；增强 for 循环编译后就是迭代器遍历；switch 支持 String 实质是先用 hashCode 再 equals 判断；try-with-resources 是编译成 try-catch-finally 并自动调用 close；Lambda 通过 invokedynamic 实现，不生成内部类文件。"

> "面试中容易被问到的语法糖陷阱：第一，自动装箱时的缓存池问题（`Integer.valueOf(-128~127)` 走缓存，用 `==` 比较可能不符合预期）；第二，泛型擦除导致运行时无法获取泛型类型（除了通过反射获取带泛型的方法签名等特殊情况）；第三，Lambda 和匿名内部类对 `this` 的引用不同——Lambda 的 `this` 指向外部类，匿名内部类的 `this` 指向自身。"

> 核心记忆：**"语法糖是写给编译器看的，编译后还原成基础语法，不影响 JVM 执行逻辑。"**

## 4. 记忆口诀

> **装箱拆箱 valueOf，泛型擦除全变 Object；for-each 就是迭代器，switch 字符串先 hashCode；可变参数转数组，try-resource 自动 close；Lambda 那叫 invokedynamic，语法糖代码真简洁。**
