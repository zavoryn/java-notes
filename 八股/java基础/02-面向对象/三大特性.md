# Java 三大特性：封装、继承、多态

## 一、封装（Encapsulation）

### 核心概念

把数据和方法包装在类中，对外隐藏内部实现细节，只暴露必要的操作接口。

### 实现方式

- 用 `private` 修饰成员变量，外界无法直接访问
- 提供 `public` 的 getter / setter 方法控制访问
- 权限修饰符逐步放开可见性：

| 修饰符 | 本类 | 同包 | 子类 | 任意 |
|---|---|---|---|---|
| `private` | Y | N | N | N |
| `default`（不写） | Y | Y | N | N |
| `protected` | Y | Y | Y | N |
| `public` | Y | Y | Y | Y |

> [!tip] 封装的本质
> **封装不是 getter/setter，是把数据的修改权从调用方手里收回来。** `person.age = -5` 你拦不住；`person.setAge(-5)` 你可以在 setter 里校验拦截。权限修饰符（private → public）是这个控制权的四个档位。

### 目的

- 保护数据完整性（setter 中可加校验逻辑）
- 降低耦合度（内部实现改变，调用方无感知）

### 示例

```java
public class Person {
    private String name;  // 私有字段，外部不可直接访问
    private int age;

    // 对外暴露的 getter
    public String getName() {
        return name;
    }

    // setter 中可加入校验逻辑
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("年龄不合法");
        }
        this.age = age;
    }
}
```

## 二、继承（Inheritance）

### 核心概念

子类通过 `extends` 关键字继承父类，自动获得父类的字段和方法（除构造方法和 `private` 成员）。

### 关键规则

> [!important] Java 继承规则
> - **单继承**：一个子类只能有一个直接父类
> - 子类**不继承**父类的构造方法（必须自己写并通过 `super(...)` 调用）
> - `private` 成员子类拿不到
> - 子类可以**重写**父类方法（不能重写 `private` / `static` / `final` 方法）
> - 通过 `super` 关键字调用父类构造方法和被覆盖的方法

### 示例

```java
public class Animal {
    private String name;        // 子类不可直接访问
    protected String species;   // 子类可访问

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating");
    }
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 必须调用父类构造方法
    }

    @Override
    public void eat() {
        super.eat();  // 调用父类被覆盖的方法（可选）
        System.out.println("Dog eats bones");
    }
}
```

> [!warning]
> 字段没有多态！继承链中如果子类定义了与父类同名的字段，子类的字段会**隐藏**父类字段，但通过父类引用访问时看到的是父类的字段。

## 三、多态（Polymorphism）-- 面试重点

### 本质

同一个方法调用，**不同对象**表现出**不同行为**。本质是**父类引用指向子类对象**。

### 实现三条件

1. 有**继承**或**实现**关系
2. 子类**重写**父类方法
3. **父类引用**指向**子类对象**

### 编译时 vs 运行时

> [!tip] 经典口诀
> **编译看左边，运行看右边。**

```java
Animal a = new Dog();  // 父类引用指向子类对象
a.eat();               // 编译期检查 Animal 是否有 eat()
                       // 运行期实际执行 Dog 的 eat()
```

### 详细说明

- **编译时**（静态绑定）：看引用类型（左边 `Animal`），检查方法是否存在
- **运行时**（动态绑定 / 运行时多态）：看实际对象类型（右边 `Dog`），执行子类重写后的方法

### 关键注意点

> [!warning] 字段没有多态
> 多态**只适用于方法**，不适用于字段。字段的访问**只看引用类型（左边）**，不看实际对象（右边）。

```java
class Parent {
    int x = 10;
    public void print() { System.out.println("Parent"); }
}
class Child extends Parent {
    int x = 20;
    @Override
    public void print() { System.out.println("Child"); }
}

Parent p = new Child();
System.out.println(p.x);  // 输出 10（字段看左边 Parent）
p.print();                // 输出 "Child"（方法看右边 Child）
```

### 多态的好处

- **可扩展性**：新增子类不需要修改使用多态的代码
- **可替换性**：符合里氏替换原则（子类可以替换父类）
- **解耦**：面向接口/父类编程，降低耦合

## 面试回答框架（三段式）

> [!info] 问：说说 Java 面向对象三大特性？

**① 一句话定位：** Java 面向对象有三大特性——封装隐藏细节、继承复用代码、多态灵活扩展。

**② 核心展开（挑 2-3 个深度点）：**

封装的核心是把数据的**修改权收回到类自己手里**。用 private 隐藏字段，通过 setter 做校验——这和权限修饰符的设计是一个思路，从 private 到 public 逐步放宽控制。

继承用 extends 实现代码复用，Java 单继承。一个很多人忽略的细节：**字段不参与多态**。`Parent p = new Child(); p.x` 访问的是 Parent 的 x，因为字段绑定是编译期的事，只有方法走动态分派。

多态的底层是**虚方法表（vtable）**。JVM 类加载时给每个类建一张表，记录每个方法实际指向哪段字节码。`Animal a = new Dog(); a.eat()` 的执行路径是：查 Animal 的 vtable → 发现 Dog 重写了 eat → 跳到 Dog.eat 的字节码。这就是「编译看左边，运行看右边」的底层原因。

**③ 落地场景：** 实际开发中，多态的典型应用是面向接口编程——`List<String> list = new ArrayList<>()`，后续换 `LinkedList` 只需改一处。框架里也很普遍：Spring 注入接口类型，运行时动态代理返回的也是多态子类。

> [!question] 追问：字段为什么不参与多态？

字节码层面，字段访问用 `getfield` 指令，直接绑定到引用类型的**固定偏移量**上，编译期就确定了。方法用 `invokevirtual` 指令走虚方法表做运行时查找。这是 JVM 规范层面的设计——如果字段也每次查表，性能代价太大。
