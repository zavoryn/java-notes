# 重写（Override）与重载（Overload）

## 一、核心对比表格

> [!tip] 一表胜千言

| 对比维度     | 重载（Overload）     | 重写（Override）                           |
| -------- | ---------------- | -------------------------------------- |
| **发生位置** | 同一个类中            | 父子类之间                                  |
| **方法签名** | 方法名相同，**参数列表不同** | 方法名、参数列表**完全相同**                       |
| **返回类型** | 可以不同，但不能仅靠返回值区分  | 返回值类型相同或为其子类（协变返回）                     |
| **访问权限** | 无限制              | 不能比父类更严格（只能放大不能缩小）                     |
| **异常**   | 无限制              | 不能抛出比父类更宽泛的 checked 异常                 |
| **多态类型** | 编译时多态（静态绑定）      | 运行时多态（动态绑定）                            |
| **修饰符**  | 无限制              | `static` / `private` / `final` 方法不能被重写 |
| **构造方法** | 构造方法可以重载         | 构造方法不能被重写                              |

## 二、重载（Overload）

### 定义

在**同一个类**中，方法名相同但**参数列表不同**（参数类型、个数、顺序不同），实现不同的功能。

### 核心规则

- 方法名必须相同
- 参数列表必须不同（类型不同 / 个数不同 / 顺序不同）
- **不能只靠返回值区分**（编译报错）
- 访问修饰符、异常声明可以随意

### 示例

```java
public class Calculator {
    // 参数个数不同
    public int add(int a) {
        return a;
    }

    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    // 以下编译报错：无法仅靠返回值区分
    // public long add(int a, int b) { return (long)(a + b); }
}
```

### 本质

**编译时多态**（静态绑定）。重载的底层是 javac 在编译时根据传入参数的静态类型和个数，计算出方法的描述符（参数类型+返回值类型），直接写入字节码。**不能只靠返回值区分**的原因是：调用处可能不接收返回值——`calc.add(1, 2);` 编译器无法确定你到底是调 `int add(int,int)` 还是 `long add(int,int)`。

## 三、重写（Override）

### 定义

在**子类**中对**父类**已有的方法重新实现。方法签名（方法名 + 参数列表）必须完全一致。

### 五大核心规则

> [!important] 面试考点：重写五大约束

1. **方法签名一致**：方法名、参数列表必须完全相同
2. **访问权限不能缩小**：子类方法的访问权限 >= 父类方法（如父类 `protected`，子类可以是 `protected` 或 `public`，不能是 `private`）
3. **异常不能扩大**：子类抛出的 checked 异常不能超出父类声明的范围（可以少抛或不抛）
4. **返回类型协变**：返回值可以是父类方法返回类型的子类（JDK5+）
5. **不能重写的特殊情况**：`static` 方法（静态方法属于类，不存在多态）、`private` 方法（不可见）、`final` 方法（显式禁止）

### 示例

```java
class Animal {
    public Animal makeSound() throws Exception {
        System.out.println("Animal sound");
        return this;
    }
}

class Dog extends Animal {
    @Override                              // 用 @Override 让编译器帮你检查
    public Dog makeSound() /* 不抛异常，符合"不扩大"原则 */ {
        System.out.println("Woof!");
        return this;  // 协变返回：Dog 是 Animal 的子类
    }
}
```

### @Override 注解的作用

- 告诉编译器这是对父类方法的重写
- 如果签名写错（如参数不对），编译器会报错，**防手误**
- 面试时可以说："@Override 不仅是一种文档注释，更是一种编译期校验手段"

## 四、特殊场景辨析

### 场景1：静态方法不被重写，而是隐藏

```java
class Parent {
    public static void hello() { System.out.println("Parent"); }
}
class Child extends Parent {
    public static void hello() { System.out.println("Child"); }  // 这是隐藏，不是重写
}

Parent p = new Child();
p.hello();  // 输出 "Parent"——静态方法看引用类型！
```

### 场景2：private 方法不能被重写

```java
class Parent {
    private void secret() { System.out.println("Parent secret"); }
}
class Child extends Parent {
    // 这不是重写，是子类自己的独立方法（因为 private 方法对子类不可见）
    private void secret() { System.out.println("Child secret"); }
}
```

## 原理深挖

### 为什么「访问权限不能缩小」？

这是**里氏替换原则（LSP）**的要求。父类声明了 `public` 方法，意味着所有父类能出现的地方，这些方法都是可调用的。如果子类把 `public` 改成 `private`，当通过父类引用调用时，JVM 发现实际对象的方法不可访问——破坏了多态的契约。

### 为什么「异常不能扩大」？

同样是 LSP。调用方只 catch 父类声明的异常类型。子类如果抛出了父类没声明的 checked 异常，调用方根本没准备处理，程序就会非预期崩溃。

### static 方法为什么不能重写？

静态方法在字节码中用 `invokestatic` 指令调用，**直接绑定到类**，不走虚方法表。子类的"同名静态方法"根本覆盖不了父类的——JVM 层面这就是两个独立的方法，是**隐藏（hide）**而非重写。

## 面试回答框架

> [!info] 问：重载和重写的区别？

**① 一句话定位：** 重载是同一个类里同一个方法名干不同的事（编译时多态），重写是子类把父类的方法换了自己的实现（运行时多态）。

**② 核心展开：** 重载看参数列表——方法名相同、参数类型/个数/顺序不同即可，和返回值、权限修饰符无关。不能只靠返回值区分——因为调用方可以不接收返回值，编译器无法判断。

重写有五大约束：签名一致、权限不缩小、异常不扩大、返回类型可协变、static/private/final 不能重写。这五条的底层逻辑是里氏替换原则——子类在任何能替换父类的地方，行为不能违背父类的契约。

**③ 落地/踩坑：** `@Override` 注解不只是注释，是编译期校验——方法名拼错一个字母编译器就报。另外注意静态方法不存在重写——`Parent p = new Child(); p.staticMethod()` 调的是 Parent 的版本，因为静态绑定发生在编译期。
