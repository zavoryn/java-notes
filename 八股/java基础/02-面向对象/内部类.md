# 内部类（Inner Class）

> [!summary] 我的速记
> **四种：** 成员内部类、静态内部类、局部内部类、匿名内部类。
> **关键区别：** 非静态内部类持有外部引用（可能内存泄漏），静态内部类不持有。
> 匿名内部类访问外部 `final` 变量（JDK8+ effectively final 即可），是 Lambda 的前身。

## 一、四种内部类

| 类型    | 定义位置     | 修饰符          | 特点       |
| ----- | -------- | ------------ | -------- |
| 成员内部类 | 类体内、方法外  | class        | 持有外部类引用  |
| 静态内部类 | 类体内、方法外  | static class | 不持有外部类引用 |
| 局部内部类 | 方法体内     | class        | 作用域限于方法内 |
| 匿名内部类 | 方法体内/参数处 | —            | 没名字，一次性的 |

```text
public class Outer {
    class Inner {}          // 成员内部类
    static class SInner {}  // 静态内部类

    void method() {
        class Local {}      // 局部内部类
    }

    void test() {
        new Thread(new Runnable() {  // 匿名内部类
            public void run() {}
        }).start();
    }
}
```

## 二、成员内部类

### 2.1 特点

- 定义在类体内，方法外
- 持有外部类引用 `Outer.this`
- 可以访问外部类的**所有成员**（包括 private）
- 不能定义静态成员（static 变量/方法）

```java
public class Outer {
    private String name = "Outer";

    class Inner {
        // static int x = 10;        // 编译错误！成员内部类不能有静态声明
        void print() {
            System.out.println(Outer.this.name);  // 访问外部类成员
        }
    }
}

// 创建
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
```

### 2.2 字节码层面

```text
Outer.class        → 外部类
Outer$Inner.class  → 内部类（$ 分隔符）
```

## 三、静态内部类

```java
public class Outer {
    private static String name = "Outer";

    static class StaticInner {
        void print() {
            System.out.println(name);           // 只能访问外部类静态成员
            // System.out.println(Outer.this.name); // 编译错误！没有外部类引用
        }
    }
}

// 创建 — 不需要外部类实例
Outer.StaticInner si = new Outer.StaticInner();
```

### 静态内部类 vs 成员内部类

| 比较维度     | 静态内部类                | 成员内部类               |
| -------- | -------------------- | ------------------- |
| 持有外部引用   | 不持有                  | 持有（`Outer.this`）    |
| 创建方式     | `new Outer.SInner()` | `outer.new Inner()` |
| 访问外部实例成员 | 不能                   | 可以                  |
| 访问外部静态成员 | 可以                   | 可以                  |
| 定义静态成员   | 可以                   | 不能                  |
| 内存泄漏风险   | 无                    | 有（持外部引用，GC 不回收）     |

> [!tip] 优先使用静态内部类，除非确实需要访问外部实例成员。可以避免内存泄漏。

## 四、局部内部类

```java
public void test() {
    final int num = 10;           // JDK8+ 可以不加 final，自动有效 final

    class Local {
        void print() {
            System.out.println(num);   // 只能访问 final/有效 final 的局部变量
        }
    }

    new Local().print();
}
```

> [!abstract] 局部内部类访问的局部变量必须是 **final 或 effectively final**（JDK8+），因为局部变量的生命周期和内部类对象不一致，内部类实际上复制了一份变量。

## 五、匿名内部类

### 5.1 基本用法

```java
// 使用场景1：事件监听
button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("按钮被点击");
    }
});

// 使用场景2：线程创建
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("新线程运行");
    }
}).start();

// 使用场景3：一步初始化
List<String> list = new ArrayList<String>() {{
    add("a"); add("b"); add("c");   // 双括号初始化（匿名子类 + 实例代码块）
}};
```

### 5.2 Lambda 替代（JDK8+）

```java
// JDK8+ Lambda 等价于匿名内部类（函数式接口）
new Thread(() -> System.out.println("Hello")).start();
button.addActionListener(e -> System.out.println("Clicked"));
```

> [!tip] Lambda 不等于匿名内部类的语法糖，Lambda 在字节码层面是通过 `invokedynamic` + 方法句柄实现的，不会生成 `$1.class` 文件，性能更好。

## 六、为什么内部类能访问外部类 private 成员？

```java
public class Outer {
    private int data = 100;

    class Inner {
        void print() {
            System.out.println(data);   // 直接访问 private！
        }
    }
}
```

> [!abstract] 编译后，编译器会为外部类自动生成 `access$xxx` 静态桥接方法：
>
> ```java
> // 编译器生成的方法（在 Outer.class 中）
> static int access$000(Outer o) {
>     return o.data;
> }
> ```
>
> 内部类实际上是通过 `Outer.access$000(this$0)` 间接访问外部类的私有成员。这个"后门"虽然能被反射调用，但**有包访问权限保护**，跨包不可访问。

## 七、内存泄漏风险

```java
public class MainActivity {
    // 危险写法！匿名内部类持有 Activity 引用
    Handler handler = new Handler() {
        @Override
        public void handleMessage(Message msg) {
            // 如果这是延迟消息，Activity 已关闭但 Handler 还没执行完
            // → Activity 无法被 GC → 内存泄漏
        }
    };
}
```

> [!danger] **非静态内部类（含匿名内部类）持有外部类引用**。如果内部类对象存活时间比外部类长，外部类就没办法被 GC，造成内存泄漏。解决方案：改用**静态内部类 + WeakReference**。

```java
// 安全写法
static class SafeHandler extends Handler {
    private final WeakReference<MainActivity> ref;
    SafeHandler(MainActivity activity) {
        ref = new WeakReference<>(activity);
    }
    @Override
    public void handleMessage(Message msg) {
        MainActivity activity = ref.get();
        if (activity != null) {
            // 安全操作
        }
    }
}
```

## 原理深挖

### 内部类访问外部 private 成员的秘密

```java
public class Outer {
    private int x = 10;

    class Inner {
        void print() { System.out.println(x); }  // 怎么能访问 private x？
    }
}
```

编译后你会发现 `Outer` 类被偷偷加了一个方法：`static int access$000(Outer o) { return o.x; }`。Inner 的 `print()` 实际编译为 `System.out.println(Outer.access$000(this.this$0))`。这个 `access$000` 是编译器生成的桥接方法，包级可见——从 JVM 的角度，`private` 始终是 `private`，编译器只是帮你绕了一下。

### 匿名内部类 vs Lambda——不是语法糖！

```java
// 匿名内部类：生成 Outer$1.class 文件
Runnable r1 = new Runnable() {
    @Override public void run() { System.out.println(this); }
};
// r1 里的 this 指向匿名内部类自身

// Lambda：不生成 .class 文件，用 invokedynamic
Runnable r2 = () -> System.out.println(this);
// r2 里的 this 指向外部类实例！
```

`this` 指向不同是最大的坑。匿名内部类有自己独立的 `this`，Lambda 的 `this` 就是写 Lambda 所在类的 `this`。所以方法引用和 Lambda 表达式在字节码层面是 `invokedynamic`，不是语法糖。

## 面试回答框架

> [!info] 问：Java 有哪些内部类？

**① 一句话定位：** 四种——成员内部类、静态内部类、局部内部类、匿名内部类。核心区分标准：是否持有外部类引用。

**② 核心展开：**

成员内部类持有 `Outer.this`，可以访问外部类所有成员含 private——底层是编译器生成了 `access$xxx` 包级桥接方法。

静态内部类不持有外部引用，本质和独立类差不多只是套了一层命名空间。创建时不需要外部类实例，优先使用。

匿名内部类是最常用的——new 接口直接写实现。但 JDK8 引入 Lambda 后，函数式接口的匿名内部类场景基本被替代。Lambda 和匿名内部类有个关键区别：`this` 指向不同——匿名内部类的 this 指向自身，Lambda 的 this 指向外部类。

**③ 踩坑经验：** 非静态内部类持有外部引用，生命周期长于外部类时会导致内存泄漏。Android 的 Handler+Activity 是经典案例——解决方式是静态内部类+WeakReference。
