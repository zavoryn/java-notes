# Redis 总览

> [!summary] 模块速记
> **Redis = 单线程快（内存+epoll）+ 数据结构丰富（SDS/跳表）+ 持久化安全（RDB+AOF）+ 高可用完整（主从→哨兵→集群）+ 功能生态全（分布式锁/Stream/缓存策略）。**

---

## 模块知识地图

```
Redis
  │
  ├── 01. 数据结构与线程模型
  │     ├── 5 大类型底层：SDS、quicklist、skiplist、intset、ziplist
  │     ├── 跳表原理（高速公路比喻）— ZSet 的核心
  │     ├── 渐进式 rehash
  │     └── 线程模型：单线程+IO多路复用、6.0 IO多线程
  │
  ├── 02. 持久化
  │     ├── RDB：快照、bgsave、COW 机制
  │     ├── AOF：日志、三种刷盘策略、AOF 重写
  │     └── 混合持久化（4.0+）
  │
  ├── 03. 缓存策略
  │     ├── 缓存模式：Cache Aside / Read Through / Write Behind
  │     ├── 穿透/击穿/雪崩：区别 + 解法
  │     └── 过期策略（惰性+定期）+ 8 种淘汰策略
  │
  └── 04. 分布式与高可用
        ├── 分布式锁：SET NX EX → Lua → 看门狗 → RedLock
        ├── 高可用：主从 → 哨兵 → 集群 → 一致性哈希
        ├── 事务：MULTI/EXEC、WATCH 乐观锁、Lua
        ├── 消息队列：List → Pub/Sub → Stream
        ├── 内存管理：jemalloc
        └── Redis vs Memcached
```

---

## 三大问题 → 解决方案

| 问题 | 场景 | 方案 |
|------|------|------|
| **缓存穿透** | 查不存在的数据 | 布隆过滤器 + 缓存空值 |
| **缓存击穿** | 热点 key 过期 | 互斥锁 + 逻辑过期 |
| **缓存雪崩** | 大量 key 同时过期 | 随机过期 + 高可用 + 限流 |
