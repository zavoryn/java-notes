### Q1：大模型（LLMs）具有什么优缺点？
> 关联：[Agent/AI](../../Agent/AI/)

**答案：**

优点：
- 可以利用大量的无标注数据来训练一个通用的模型，然后再用少量的有标注数据来微调模型，减少数据标注的成本和时间，提高泛化能力。
- 可以利用生成式人工智能技术来产生新颖和有价值的内容（图像、文本、音乐等）。
- 可以利用涌现能力（Emergent Capabilities）来完成数学应用题、常识推理、符号操作等任务。

缺点：
- 需要消耗大量的计算资源和存储资源来训练和运行，增加经济和环境的负担。
- 需要面对数据质量和安全性的问题，例如数据偏见、数据泄露、数据滥用等。
- 需要考虑可解释性、可靠性、可持续性等方面的挑战。

### Q2：Agent 和 Prompt Chain 有什么本质区别？
> 关联：[Agent/AI](../../Agent/AI/)

**答案：** Prompt Chain 的拓扑与顺序主要由工程侧固定；Agent 在运行时在动作空间中做选择，并依赖 Observation 更新信念，适合输入与路径不确定的任务。二者可结合：链负责稳定流程，Agent 负责链内某段的灵活分支。

### Q3：ChatBot 加上插件是不是就变成 Agent 了？
> 关联：[Agent/AI](../../Agent/AI/)

**答案：** 不一定。若插件调用由固定规则触发（例如关键词路由），更像「带工具的 Bot」。若由模型在多步推理中自主选择工具与参数，并形成闭环迭代，则更贴近 Agent。关键在是否具备多步自主决策与反馈闭环。

### Q4：RAG + Chat 算不算 Agent？
> 关联：[Agent/AI](../../Agent/AI/)

**答案：** 单次检索再回答，偏「增强型 Chat」。若有多轮检索策略（查不到换查询、分解子问题、交叉验证），则具备 Agent 特征。
