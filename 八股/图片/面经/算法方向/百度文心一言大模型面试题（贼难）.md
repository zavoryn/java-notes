### 1. 3分钟自我介绍

我叫阿俊AI，学习硕士，以前主要做 Java 后端开发，比如微服务、数据接口这些。最近两年主要在做**大模型落地**：从造数据、训练模型、评测效果好坏，到把模型部署上线优化速度，全流程都做过。技术上会 PyTorch、大模型训练工具、推理加速工具，也懂 Java 后端特征服务、AB 测试。最近一个项目里我负责大模型驱动的业务报告自动生成，最后报告人工撰写耗时从小时级降到分钟级，内容完整率提升 35%，事实错误率下降 40%。

### 2. 手撕带因果掩码的多头注意力

不需要记复杂公式，记住流程就行：
1. 把输入分别变成 Q、K、V 三个向量
2. 把它们拆成好多"头"，分头算注意力
3. 算 Q 和 K 的相似度，再除以一个数防止数值太大
4. 加因果掩码：保证每个字只能看到它前面的字（看不到后面的）
5. softmax，得到每个位置对其他位置的权重
6. 用权重乘 V，得到每个位置的输出
7. 把头合并，最后做一次线性输出

**总结：** 因果掩码 = 只能往前看，不能往后看，保证生成文字按顺序来

### 3. 报告生成质量怎么自动评估？

不只用传统的 BLEU/ROUGE，而是从多个角度打分：
- **事实对不对：** 和原始资料有没有矛盾、胡说八道
- **内容全不全：** 该写的点有没有漏掉
- **读起来顺不顺：** 有没有重复、太长太短
- **合不合规：** 有没有敏感词、违规内容

**怎么自动化实现：**
- 简单的用规则检查：必填项有没有、格式对不对
- 用向量相似度：看生成内容和标准答案像不像
- 用小模型当"裁判"：按 1~5 分逐项打分
- 用推理模型判断：生成内容是否符合原文事实

**最后闭环：** 自动分数要和人工打分对齐，线上还要监控指标会不会飘。

### 4. SFT + DPO 数据怎么来？

**SFT（监督微调）：** 给模型看"问题 → 标准答案"。数据来源：公开通用指令数据 + 自己业务造的脱敏数据，混合着用，让模型学会怎么回答、用什么格式。

**DPO（偏好对齐）：** 给模型看"同一个问题，好回答 vs 差回答"。数据来源：人工标那个好那个差；从用户日志里看用户更喜欢哪个回答；让模型自己生成多个答案，再用裁判模型挑好坏。

**总结：** 先 SFT 学会怎么说话 → 再 DPO 学会说人话，说对的话。

### 5. 大模型怎么做"生成排序"？（比如推荐排序）

简单理解：让大模型给商品/内容排个序。三种常用方法：
- **直接排：** 把一堆候选塞给模型，让它直接输出排好的序号
- **逐个打分：** 对每个候选问"相关性几分？"最后按分数排序
- **两两对比：** 让模型比较 A 和 B 谁更好，反复比完再排

**工程上关键点：** 输入别太长，输出要能解析，不行就降级兜底。

### 6. 为什么要用 LLM 排序？比传统模型好在哪？

**传统模型（DIN、DeepCTR）：** 快、便宜、成熟，适合大量流量。

**LLM 优点：**
- 语义理解强：新词、冷门词都能懂
- 能处理复杂意图：比如"便宜又好用的 XX"
- 能同时兼顾相关性、多样性、规则约束
- 冷启动更好：没太多用户行为也能靠文本理解

**不是谁替代谁，而是：** 传统模型扛流量，LLM 放在最后一层做精细重排，提升效果。

### 7. LLM 部署在线推荐，速度慢怎么解决？

从三个方向优化：
- **模型本身量化（变小变快）：** KV 缓存、只输关键信息、用小模型
- **推理引擎：** 用 vLLM 这类加速工具、批量处理请求
- **架构：** LLM 只做最后重排，不做全量召回，超时就降级用传统模型兜底

### 8. 高并发（几千 QPS）怎么落地 LLM？

LLM 不可能扛全部流量，必须分层：
- **召回、粗排：** 用传统快模型，扛住大部分请求
- **精排/重排：** 只对少量候选（几十~几百个）用 LLM

**高并发保障：** 多实例部署、负载均衡、限流、降级、缓存。

**总结：** LLM 只干精细活，脏活累活交给传统模型。

### 9. 你实际用过哪些优化？

我们线上是：传统召回 → 轻量模型粗排 → LLM 对 Top50 重排。用过：vLLM 推理加速、FP8 量化、Prompt 缓存、超时降级机制。

### 10. LoRA、模型融合、Mamba 是什么？

- **LoRA：** 训练大模型时，只训练很小一部分参数，又快又省显存。推理时可以把 LoRA 合并进原模型，不影响速度
- **LoRA 融合：** 把训练好的 LoRA 和基座模型合成一个文件，方便上线部署
- **Mamba：** 一种比 Transformer 更快的序列模型，长文本更有优势，但目前还不太成熟，大多在实验阶段

### 11. Caching + Reranking 怎么做？

**标准结构：**
1. 多路召回（热门、协同、向量）
2. 轻量模型精排
3. 选出 Top 50~200，LLM 再做一次重排，兼顾语义与规则
4. 加缓存：相同请求直接返回，不用重新算
5. 不让 LLM 完全推翻传统排序，避免线上波动

### 12. 传统 CTR 快、LLM 慢，能不能蒸馏？

可以，这叫**知识蒸馏**：
1. 用效果好但慢的 LLM 给大量数据打标签
2. 训练一个又快的学生模型（双塔、DIN）去学 LLM 的排序逻辑
3. 线上只跑学生模型，速度不变，效果接近 LLM
