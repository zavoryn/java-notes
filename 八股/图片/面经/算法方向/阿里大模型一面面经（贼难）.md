# 阿里大模型一面面经（贼难）

> 来源：抖音博主「阿俊AI」
> 原文链接：https://www.douyin.com/note/7621749690208856680

### 1. 目前主流大语言模型在架构设计上有哪些异同点？

答：
**相同点：** 主流均基于 Transformer Decoder-only 架构（GPT系列、LLaMA、Qwen等），采用预训练-微调范式，使用自回归生成方式。
**不同点：** 位置编码方式（绝对/旋转/ALiBi）、Norm层位置（Pre-Norm/Post-Norm）、激活函数（GeGLU/SiLU）、并行注意力与FFN设计、是否使用MQA/GQA优化推理、上下文窗口大小等。

### 2. 介绍一下大语言模型中的注意力机制，多头相比单头注意力有何优势？

答：注意力机制通过 QK^T 计算相似度对 V 加权，动态聚焦上下文关键信息。
**多头优势：** 可并行学习多组不同语义空间的注意力权重，分别捕获语法、语义、远程依赖等多元特征，表达能力更强，泛化与建模效果更优。

### 3. 什么是大语言模型的涌现能力？目前对该现象的研究有哪些发现？

答：涌现能力是模型规模/数据量达到阈值后，突然具备小模型没有的复杂能力，如推理、数学解题、代码生成。
研究发现：能力涌现与参数量、训练数据、任务复杂度相关；遵循 Scaling Law；部分能力可通过指令微调提前激发；小模型难以复现。

### 4. 什么是 Embedding？词嵌入和句嵌入有何不同？

答：Embedding 是将离散符号映射为稠密低维向量，用于表征语义信息。
- **词嵌入**：表征单个词汇语义
- **句嵌入**：整段文本的语义向量，更侧重全局语义与上下文，常用于检索、相似度计算

### 5. 大语言模型中的 Tokenization 是如何工作的？不同模型的分词算法有何差异？

答：Tokenization 将文本切分为模型可处理的子词单元，包含预处理、分词、编码映射。
**差异：**
- BPE（GPT、LLaMA）
- WordPiece（BERT）
- Unigram（T5、多语言模型）

对稀有词、长词、多语符号的切割策略不同。

### 6. 简述大语言模型中的 Prompt Engineering 技巧。如何设计有效的提示词提升模型输出质量？

答：核心技巧：明确角色定位、清晰任务指令、给出约束格式、提供 Few-Shot 示例、使用思维链（COT）、分步拆解任务、限定知识边界、要求引用依据。
**有效设计：** 具体、结构化、可验证，减少模糊表述，引导模型按目标完成输出。

### 7. 对 Zero-Shot、Few-Shot 和 In-Context Learning，应用场景和局限性分别是什么？

答：
- **Zero-Shot：** 无示例直接推理，适合简单通用任务；局限是复杂任务效果差
- **Few-Shot：** 提供少量示例，提升任务适配性；局限是示例数有限，易受噪声干扰
- **In-Context Learning：** 利用上下文示例引导推理，无需微调；局限是依赖上下文窗口，长距离示例效果衰减

### 8. 什么是 LoRA？它在大语言模型微调中的优势和原理是什么？

答：
- **原理：** 冻结主干模型，在注意力层插入低秩矩阵适配，仅训练少量参数
- **优势：** 显存占用低、训练快、轻量化、可插拔、不易灾难性遗忘、多任务适配灵活

### 9. 大语言模型的 RLHF 训练流程是什么？它存在哪些潜在风险？

答：
- **流程：** SFT 监督微调 → 训练奖励模型 RM → PPO 强化学习对齐人类偏好
- **风险：** 奖励作弊、对齐税（能力下降）、奖励过拟合、分布偏移、灾难性遗忘、训练不稳定

### 10. 介绍一下大语言模型中的知识蒸馏，它如何用于压缩模型体积？

答：知识蒸馏以大模型为教师，小模型为学生，学习其软标签分布、特征与逻辑。可通过蒸馏大幅降低参数量与推理成本，同时保留大部分能力，实现模型轻量化部署。

### 11. DeepSeek 做了哪些优化？为什么不用 PPO，而是用 GRPO？

答：
- **优化点：** 高效训练架构、长上下文、GQA、多语言与代码能力、对齐优化
- **GRPO 相比 PPO：** 训练更稳定、样本效率更高、策略更新更温和、降低超参敏感度，适合大规模对齐训练

### 12. 对比大语言模型的增量推理和传统推理方式。增量推理的优势和实现难点是什么？

答：
- **优势：** 复用历史 KV Cache，仅计算新 Token，显著提速，降低计算量
- **难点：** KV Cache 显存占用随长度膨胀、长文本管理复杂、并行生成受限、部分架构兼容性差、缓存失效与更新逻辑复杂

### 13. 大语言模型在推理时出现幻觉现象的原因是什么？有哪些缓解方法？

答：
- **原因：** 训练数据噪声、知识过时、模型拟合文本分布而非事实、不确定性建模不足
- **缓解：** RAG 检索增强、事实校验、降低温度、引用溯源、指令约束、长文本结构化、对齐微调

### 14. 大语言模型对长文本处理能力有限，目前有哪些技术可以缓解这一问题？

答：旋转位置编码、ALiBi、滑动窗口注意力、稀疏注意力、MQA/GQA、分块摘要、RAG 检索、动态窗口扩展、LongLoRA 等微调方案。

### 15. 什么是大语言模型的上下文窗口？扩展上下文窗口对模型性能有何影响？

答：上下文窗口是模型一次能处理的最大 Token 数量。扩展后可处理长文档，但会提升推理显存与延迟，可能导致远程注意力下降，未经微调直接扩窗会损害语义建模。

### 16. 目前多模态大语言模型是如何融合文本与图像信息的？

答：图像经视觉编码器提取特征 → 通过投影层对齐到文本语义空间 → 与文本 Embedding 拼接输入 LLM → 跨模态注意力交互实现融合生成。

### 17. 如何评估大语言模型的性能？常见的评测指标和基准数据集有哪些？

答：
- **指标：** 困惑度 PPL、准确率、BLEU、ROUGE、MMLU、GSM8K、HumanEval 等
- **基准：** MMLU、C-Eval、CSMKB、BBH、HumanEval、MBPP、MIT-Bench、TruthfulQA

### 18. 单个 LLM 能否帮助用户完成完整行程规划，比如从出行到酒店都订好票？

答：纯 LLM 无法直接订票，需结合 Function Calling 调用外部 API（票务、酒店、地图）。LLM 负责意图理解、规划编排、对话交互，工具负责实际执行，可实现端到端行程规划。
