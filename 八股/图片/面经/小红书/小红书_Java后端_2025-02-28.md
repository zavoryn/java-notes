## 面试问题

### Q1. 在项目中处理并发问题的经验？什么情况下会加行锁？

- 低并发：`synchronized` 简单粗暴，性能比 `ReentrantLock` 还好（JVM底层有自旋优化）
- 高并发：`ReentrantLock` 手动控制，设置超时时间避免死锁
- 秒杀场景：乐观锁+版本号，配合Redis预减库存

> 关联：[juc并发编程](../../juc并发编程/)

### Q2. MVCC和乐观锁的关系

- 每个事务都有专属快照
- 更新时检查版本号
- 如果发现别人改过就重试（类似Git冲突解决）
- 小红书的帖子点赞系统就用这个，读多写少不怕冲突

> 关联：[mysql](../../mysql/)

### Q3. 限流算法在工作中怎么选型？

| 算法 | 适用场景 | 例子 |

| 漏桶 | 稳定流速 | API每分钟1000次调用 |
| 令牌桶 | 允许突发 | 双11秒杀开场流量洪峰 |
| 滑动窗口 | 精准控制 | 实时风控系统 |

> 关联：[微服务](../../微服务/)、[redis](../../redis/)

### Q4. Redis持久化怎么保证数据安全？

- **RDB**（定期存折）：整点快照，恢复快但可能丢几分钟数据
- **AOF**（流水账）：每笔操作都记账，最多丢1秒数据
- 小红书把两者都打开，`appendfsync everysec` 平衡性能与安全

> 关联：[redis](../../redis/)

### Q5. 遇到过频繁Full GC吗？怎么排查的？

1. **内存泄漏**：错误缓存策略导致对象堆积（用MAT找到真凶）
2. **元空间不足**：动态生成类太多
3. **大对象**：有个1MB的JSON反复序列化
- 解决：`-Xmx` 调大 + 改用堆外缓存

> 关联：[jvm虚拟机](../../jvm虚拟机/)

### Q6. Spring用了哪些设计模式让你眼前一亮？

- **代理模式**：AOP切面动态增强
- **单例模式**：Bean默认作用域省资源
- **模板方法**：JdbcTemplate消除重复代码
- **观察者**：事件监听机制超解耦

> 关联：[spring](../../spring/)、[设计模式](../../设计模式/)

### Q7. 手写LRU缓存

```python
class LRUCache:
    def __init__(self, capacity: int):
        self.cache = OrderedDict()
        self.cap = capacity

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        else:
            if len(self.cache) >= self.cap:
                self.cache.popitem(last=False)
        self.cache[key] = value
```

核心思路：字典快速查找 + 双向链表维护访问顺序，时间复杂度O(1)
