# 腾讯Agent面试题（好难）

> 来源：抖音博主「阿俊AI」
> 注：本帖为面试题系列合集的Q11-18

### 11. MCP协议是什么？

MCP，Model Context Protocol。你可以把它理解成模型应用和外部能力之间的标准化连接协议。

注意，是"标准化连接协议"，不是"又一种函数调用格式"。很多人把MCP理解成Function Calling的替代品，这认知就太浅了。

MCP解决的是三个问题：
- 外部能力怎么统一暴露
- 外部能力怎么统一发现
- 外部能力怎么统一调用

它一般覆盖三类能力：
- **Tools：** 可执行能力
- **Resources：** 可读资源
- **Prompts：** 可复用提示模板

一句话概括：MCP是大模型时代的能力接入协议，不是某个模型厂商私有的函数格式小把戏。

### 12. MCP协议体现在什么地方？

MCP从不是虚头巴脑的概念，真身就三件事：

1. **清单声明：** Server通过Manifest暴露tools、resources、prompts，把能力亮明白。
2. **传输规约：** stdio、流式HTTP做载体，JSON-RPC完成消息交互，干净利落。
3. **执行闭环：** Client拉取工具→转模型调用格式→模型返回指令→执行工具→回传结果，全链路标准化。

市面上Cursor、Claude Desktop、各类集成MCP的IDE，界面上的配置项、日志里的tools/list、tools/call，全是这套协议的表层皮。懂行的一眼就能看穿底层逻辑，没任何花活。

### 13. LangChain4j有用到MCP吗？

LangChain4j有一套体系，但要说它用到了MCP，纯属无稽之谈。LangChain4j不过是为了方便普通开发者集成大模型而搭建的框架，专注于提供通用的工具与流程。而MCP这种涉及模型核心控制的协议，岂是LangChain4j这种面向大众普及的框架能驾驭的。两者根本不在一个技术层面上，LangChain4j还远远够不上MCP的门槛。

### 14. Function Calling和MCP有什么区别？

先把它们放在不同抽象层看：

**Function Calling**解决的是：模型如何表达"我要调用某个函数"的问题，重点在模型输出格式，比如函数名、参数JSON、调用结构化结果。

**MCP**解决的是：外部世界如何把工具、资源、提示能力用统一协议暴露给模型应用，重点在系统集成和能力治理，不在模型输出本身。

所以最精炼的区别是：
- **Function Calling：** 偏模型接口层
- **MCP：** 偏系统集成层

或者更狠一点说：Function Calling解决"模型怎么喊"，MCP解决"工具世界怎么被组织起来"。

它们不冲突，甚至经常配合：模型用Function Calling决策要调用什么，底层再通过MCP去接真正的工具服务。这才是完整答案。

### 15. 如何实现使用大模型调自己的接口？

首先，要清晰定义自己接口的输入输出规范，这得精准到极致，容不得半点模糊。然后，借助Function Calling机制，将接口的详细描述嵌入到与大模型交互的提示中，得让模型明白：何时该调这个接口、如何根据当前输入构造正确的调用参数。当模型判断需要调用时，按照预设格式生成请求，执行接口调用。

这看似简单，实则需要对模型理解和接口设计的深度把握，不是一般人能轻易做到的。

### 16. 大模型用的什么引擎？

主流大模型大多以Transformer架构作为核心。

Transformer凭借自注意力机制（Self-Attention），打破了传统序列处理的桎梏，实现了并行计算，极大提升了模型对上下文的理解与捕获能力，让模型在处理复杂任务时游刃有余。

这是一场架构上的革命，引领大模型走向新的高度。

### 17. 为什么要使用阿里云百炼？阿里云百炼有啥好处？

阿里云百炼不过是众多大模型服务中的一个。好处无非是：

1. 经过大量数据预训练，在常见自然语言理解和业务任务上有一定表现
2. 提供了还算稳定的API，供开发者调用
3. 其背后依托阿里云的云计算能力，能保证一定的服务稳定性

但说到底，这只是适合那些没有能力自己构建与优化模型的人。真正的技术高手，追求的是对模型的绝对掌控。

### 18. AI方向在项目或者自己怎么使用？

在项目中：
1. 先明确目标，要有超越常规的眼光和判断力
2. 选模型——要么有资源去精心挑选最合适的，要么有能力自己搭建
3. 对于文本任务，提示工程要做到出神入化，挖掘模型潜力
4. 图像任务，数据标注要精准无误，训练过程要精心雕琢

个人使用AI，别只满足于简单应用，要深入探索模型原理，尝试基于开源框架搭建独特应用，提升自己的技术境界。
