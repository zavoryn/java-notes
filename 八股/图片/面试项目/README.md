## 面试项目讲述框架

面试官通常会按以下顺序提问，每个项目按这个结构准备：

### 1. 项目介绍（1-2分钟）
- 业务背景：解决什么问题
- 系统规模：用户量、QPS、数据量
- 你的角色：负责哪些模块

### 2. 技术架构
- 整体架构图（微服务/单体、部署方式）
- 核心技术选型及原因
- 数据库/缓存/消息队列的使用

### 3. 技术难点（重点）
- 遇到的具体问题
- 分析过程
- 解决方案（为什么选这个方案）
- 最终效果（量化指标最好）

### 4. 项目反思
- 如果重新做会怎么改进
- 学到了什么
