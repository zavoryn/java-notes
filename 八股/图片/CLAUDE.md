# Java 八股文知识库

## 项目说明
存放 Java 面试八股文相关的题目和答案，按知识点分类组织。

## 知识来源
1. **飞书云文档** - 主要来源，通过 lark-cli 获取
   - 微服务文档：https://my.feishu.cn/wiki/FLS2wDVlmiJUepknxyRc4P3Enjd
   - Java八股复习计划（多维表格）：https://my.feishu.cn/base/ZyVJbwHEiaYUupsx7akca4BRnVg
   - 其他八股文档（计算机网络、MySQL、多线程等）也在云文档中
2. **用户直接提供** - 用户提供的图片、文字等内容

## 目录结构

### 知识点模块（按技术分类）
每个知识点一个子文件夹，内含 `.md` 文件：
- java基础/
- juc并发编程/
- jvm虚拟机/
- spring/
- mysql/
- redis/
- 消息队列/
- 设计模式/
- 计算机网络/
- 操作系统/
- 微服务/

### 开源经历
- 开源经历/ - 存放开源贡献的面试准备材料
  - `开源经历.md` - PR 描述、面试问答准备、简历版描述
  - 关联到对应知识点模块（如微服务/Nacos 相关）

### 面经（按面试场次分类）
- 面经/ - 存放每次面试的完整记录
  - 每篇面经一个 MD 文件，文件名格式：`公司_岗位_日期.md`
  - 面经中的每个问题必须标注关联的知识点模块（用相对链接）
  - 如果面经中出现了知识点模块中未覆盖的新问题，同步补充到对应模块

### 面试项目（按项目分类）
- 面试项目/ - 存放项目经历的面试准备材料
  - `README.md` - 项目总览 + 面试讲述框架
  - 每个项目一个 MD 文件，包含：项目概述、技术栈、架构、技术难点、亮点、面试可能的问题
  - 简历版项目描述（精炼版，可直接放入简历）
  - 项目中的技术点关联到对应知识点模块

### 博客
- 博客/ - 存放技术博客文章
  - 每篇文章一个 MD 文件
  - 文件头标注来源链接（掘金/CSDN）和发布时间
  - 文章内容关联到对应知识点模块

## 工作方式
- 从飞书云文档获取内容时，忠实提取原文，不做额外展开或编造
- 从图片提取文字时，逐字转录，保持原排版
- 从网页抓取内容时，忠实提取原文，在文档末尾标注来源链接
- 用户直接提供的内容，按分类整理到对应子文件夹
- 新增知识点时，创建对应子文件夹和 MD 文件

## 面经关联规则
新增面经时，必须遵守以下规则：
1. 每个面试问题用 `###` 标题列出
2. 在问题标题下方用 `> 关联：[模块名](相对路径)` 标注关联的知识点
3. 如果该问题在知识点模块中有对应内容，直接链接；如果没有，先补充知识点再链接
4. 面经文件末尾统一列出本次面试涉及的所有知识点模块汇总

## 面试项目规则
新增或更新项目时，必须遵守以下规则：
1. 每个项目一个 MD 文件，文件名格式：`项目名.md`
2. 按统一模板整理：项目概述 → 技术栈 → 系统架构 → 核心功能 → 技术难点与方案 → 亮点 → 面试问答准备
3. 项目中的技术点用 `> 关联：[模块名](相对路径)` 链接到知识点模块
4. 提供简历版描述（200字以内精炼版）
5. 更新 `面试项目/README.md` 的项目总览表
