# JVM 虚拟机

## 垃圾回收（GC）★★★★★

1. GC的作用区域（堆和方法区）
2. 判断对象可回收的算法（引用计数法、可达性分析算法）及引用类型
3. 常见的垃圾收集算法（标记-清除、标记-复制、标记-整理、分代收集）
4. 垃圾收集器（SerialGC、ParallelGC、CMS、G1、ZGC）的特点与适用场景
5. CMS收集器的工作流程（初始标记、并发标记、重新标记、并发清除）及缺点
6. G1收集器的核心特点（区域化分代式、Mixed GC、可预测停顿时间）
7. 内存泄漏的定义、常见场景

## JVM内存模型 ★★★★

1. JVM整体架构（类加载子系统、运行时数据区、执行引擎、本地方法接口）
2. 运行时数据区细分（程序计数器、虚拟机栈、本地方法栈、堆、方法区）及作用
3. 堆内存的细分（新生代：Eden、S0、S1；老年代；永久代/元空间）
4. 各内存区域的OOM异常及产生原因
5. 栈帧的组成（局部变量表、操作数栈、动态链接、方法返回地址）（概率低，选择性学习）

## 类加载机制 ★★★

1. 类加载的生命周期（加载、验证、准备、解析、初始化、使用、卸载）（概率低，选择性学习）
2. 类加载器的分类（四种）（概率低，选择性学习）
3. 双亲委派模型的原理、优势及打破方式
4. 类初始化的触发条件（主动引用 vs 被动引用）（概率低，选择性学习）

## 性能调优 ★★★★ 有的面试官喜欢问

1. JVM调优的流程与目标（降低GC频率、减少停顿时间等）
2. 常用JVM调优工具（jps、jstat、jinfo、jmap、jstack、jconsole、visualvm）
3. 常见调优场景（堆内存调整、垃圾收集器选择、线程池配合等）

## 执行引擎 ★（概率低，选择性学习）

1. JVM执行字节码的方式（解释执行、即时编译）
2. JIT编译的原理及优势
3. 方法的调用机制（静态分派、动态分派、单分派、多分派）
